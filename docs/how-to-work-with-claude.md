# როგორ ვიმუშაოთ Claude-თან

## რა არის Claude Code?

Claude Code არის AI ასისტენტი, რომელიც შენს პროექტში პირდაპირ მუშაობს.
შენ აღწერ რა გინდა ჩვეულებრივი ენით (ქართულად ან ინგლისურად),
და Claude შეასრულებს — კოდს დაწერს, ფაილებს შეცვლის, ტესტებს გაუშვებს.

შენ არ გჭირდება კოდის ცოდნა. უბრალოდ აღწერე რა გინდა.

## ძირითადი ბრძანებები

### რაღაცის დამატება ან შეცვლა

```
"დაამატე ახალი endpoint სერვერზე რომ ლექციის სტატუსი დააბრუნოს"

"შეცვალე WhatsApp შეტყობინების ფორმატი — დაამატე ლექციის ნომერი"

"გააკეთე ახალი ფუნქცია რომ ლექციის ხანგრძლივობა გამოთვალოს"
```

### ტესტირება და შემოწმება

```
"გაუშვი ტესტები და მაჩვენე შედეგი"

"შეამოწმე სერვერი მუშაობს თუ არა"

"გატესტე Zoom webhook — მუშაობს სწორად?"
```

### ინფორმაციის მიღება

```
"შეამოწმე რა ცვლილებები გავაკეთე ბოლო სესიაზე"

"მაჩვენე როგორ მუშაობს ლექციის დამუშავების პროცესი"

"რა არის ბოლო 5 კომიტი?"
```

### ანალიზი და დიაგნოსტიკა

```
"წაიკითხე ლექციის ვიდეო და გააანალიზე"

"რატომ არ მუშაობს WhatsApp ასისტენტი?"

"შეამოწმე Google Drive-ზე ატვირთვა მუშაობს თუ არა"
```

## გადაუდებელი ბრძანებები

თუ რაღაც გაფუჭდა ან ცვლილება არ მოგეწონა:

| რას ამბობ | რას აკეთებს Claude |
|-----------|---------------------|
| "გააუქმე ბოლო ცვლილება" | აბრუნებს პროექტს წინა მდგომარეობაზე |
| "რაღაც გაფუჭდა, გაასწორე" | ავტომატურად პოულობს პრობლემას და ასწორებს |
| "დააბრუნე ბოლო მომუშავე ვერსია" | აღადგენს ბოლო სამუშაო ვერსიას |
| "მაჩვენე რა შეცვალე" | გასაგებ ენაზე ხსნის ყველა ცვლილებას |
| "გაჩერდი, აღარაფერი შეცვალო" | აჩერებს ყველა მიმდინარე ოპერაციას |
| "თავიდან დაიწყე ბოლო შენახული წერტილიდან" | სრულად აღადგენს ბოლო save point-ს |

## რჩევები ეფექტური მუშაობისთვის

### იყავი კონკრეტული

ნაცვლად "გააუმჯობესე სერვერი"-სა, თქვი:
```
"დაამატე /status endpoint-ი რომ აჩვენოს რამდენი ლექცია დამუშავდა დღეს"
```

### ნახე შედეგი

Claude ყოველი ცვლილების შემდეგ გეკითხება:
"ნახე შედეგი — კარგად გამოიყურება?"

- თუ კარგია, თქვი **"კი"** ან **"კარგია"**
- თუ რაღაც არ მოგწონს, აღწერე რა შეცვალო

### ნაბიჯ-ნაბიჯ

დიდი ცვლილებებისთვის, Claude თვითონ ეკითხება ნებართვას.
თუ ბევრ ფაილს ცვლის, ჯერ გეტყვის რას აპირებს.

მაგალითი:
```
Claude: "მჭირდება 5 ფაილის შეცვლა. ეს გავლენას მოახდენს
ლექციის დამუშავებასა და WhatsApp შეტყობინებებზე.
გავაგრძელო?"

შენ: "კი, გააგრძელე"
```

## რას აკეთებს Claude ავტომატურად

შენ არ გჭირდება ამაზე ფიქრი — Claude თვითონ აკეთებს:

- **შენახვა (checkpoints)**: ყოველი მნიშვნელოვანი ცვლილების შემდეგ
  ავტომატურად ინახავს პროექტის მდგომარეობას
- **ტესტირება**: კოდის ცვლილების შემდეგ ავტომატურად უშვებს ტესტებს
- **უსაფრთხოება**: ამოწმებს რომ პაროლები და API გასაღებები
  არ მოხვდეს კოდში
- **შეცდომების გასწორება**: თუ რაღაც გაფუჭდა, ჯერ აღადგენს
  მომუშავე ვერსიას და მერე ასწორებს

## ხშირი სცენარები

### ახალი ლექციის ფოლდერის შექმნა

```
"შექმენი მე-8 ლექციის ფოლდერი პირველი ჯგუფისთვის Google Drive-ზე"
```

### WhatsApp ასისტენტის ტესტირება

```
"გააგზავნე სატესტო შეკითხვა WhatsApp ასისტენტთან და მაჩვენე პასუხი"
```

### ლექციის ხელით დამუშავება

```
"დაამუშავე ბოლო ლექციის ჩანაწერი — ატვირთე Drive-ზე და გააგზავნე
შეტყობინება ჯგუფში"
```

### პრობლემის დიაგნოსტიკა

```
"სერვერი არ პასუხობს — შეამოწმე რა ხდება"
```

### სტატისტიკის ნახვა

```
"მაჩვენე რამდენი ლექცია დამუშავდა ამ თვეში ორივე ჯგუფისთვის"
```

## რა არ შეუძლია Claude-ს

- არ აქვს წვდომა Zoom-ის ან Google-ის ანგარიშზე პირდაპირ
  (მხოლოდ API-ით მუშაობს)
- ვერ ხედავს შენს ბრაუზერს ან ეკრანს (თუ screenshot არ გაუგზავნი)
- ვერ გააკეთებს ცვლილებას production-ზე შენი თანხმობის გარეშე
- ვერ წაშლის ფაილებს შენი ნებართვის გარეშე

## დახმარება

თუ არ იცი როგორ სთხოვო Claude-ს, უბრალოდ აღწერე
რა შედეგი გინდა ნახო. Claude თვითონ გაარკვევს როგორ გააკეთოს.

მაგალითი:
```
"მინდა რომ ლექციის შემდეგ ავტომატურად გაეგზავნოს
სტუდენტებს შეჯამება WhatsApp-ში"
```

Claude გეტყვის რა ნაბიჯებია საჭირო და დაიწყებს მუშაობას.
