## Per-Issue Detail

### #41 — Duplicate `/paperclip/task` (CRITICAL)
**Location:** `tools/app/paperclip_bridge.py:209` vs `tools/app/server.py:2491`.
**Root cause:** `@router.post("/paperclip/task")` in the router was identical to `@app.post("/paperclip/task")` already on the FastAPI app; FastAPI silently kept the first registration, leaving the router-side function as dead code.
**Fix:** Removed the duplicate decorator + function body in `paperclip_bridge.py`, left an explanatory comment block.
**Verify:** `test_paperclip_bridge.py` 35/35 pass.

### #42 — `time.sleep` blocks event loop (HIGH)
**Location:** WhatsApp rate limiter `tools/integrations/whatsapp_sender.py:142, 402, 451`; async call sites in `tools/app/server.py:259, 1106, 1112, 1488`.
**Root cause:** `alert_operator()` is sync and the rate limiter uses `time.sleep`. Calling it from `async def` handlers blocked the FastAPI event loop while waiting.
**Fix:** Async call sites now use `await asyncio.to_thread(alert_operator, ...)`. Added `_fire_alert` helper for sync paths that may run under an event loop.
**Verify:** `test_server.py` 97/97 pass.

### #43 — `TrustedHostMiddleware ["*"]` on Railway (HIGH)
**Location:** `tools/app/server.py:601`.
**Root cause:** Wildcard `["*"]` accepted any Host header, exposing host-header-injection class.
**Fix:** Replaced with explicit suffix wildcards (`.railway.app`, `.up.railway.app`, `.railway.internal`, `healthcheck.railway.app`) plus configured public domains.
**Verify:** Smoke import OK; functional Railway test deferred to deploy.

### #44 — PII / retention / GDPR (HIGH, **PARTIAL**)
**Location:** `tools/services/message_archive.py:453+` + `tools/app/scheduler.py:1498+`.
**Root cause:** `messages` table accumulated forever; `senders.phone_encrypted` planned but stored as empty blob; no GDPR right-to-erasure path.
**Fix (this session):**
1. `purge_old_messages(conn, days)` — daily retention sweep (default 90 days, `MESSAGE_RETENTION_DAYS` env override) deletes old messages and redacts orphaned senders.
2. `gdpr_delete_sender(conn, phone)` — single-phone erasure for Article 17 requests.
3. Scheduler job at 03:15 Tbilisi daily, 15 minutes after the 03:00 backup.
**Deferred:** Actual AES-256-GCM encryption of `phone_encrypted` blob — requires `SENDER_ENC_KEY` provisioning + key-rotation policy, both outside this session's scope.
**Verify:** Smoke test with in-memory DB exercised purge + erasure successfully.

### #45 — `/admin/reset-pipeline` orphans (HIGH)
**Location:** `tools/app/admin_routes.py:225+`, `tools/integrations/knowledge_indexer.py:909+`.
**Root cause:** Reset deleted only the state file, leaving Drive summary/report docs + uploaded video + Pinecone vectors orphaned.
**Fix:**
1. Added `delete_lecture_vectors(group, lecture, content_type=None)` to `knowledge_indexer.py`. Uses ID prefix `g{N}_l{N}_…`, chunks deletes into 1000-ID batches, best-effort.
2. `reset_pipeline` now collects `summary_doc_id`, `report_doc_id`, `drive_video_id`; moves each Drive file to trash (recoverable 30 days); calls `delete_lecture_vectors`; then unlinks state.
3. Each step independent + failure-isolated. Response includes `orphan_cleanup` counters + errors for operator visibility.
**Verify:** `test_admin_routes.py` 28/28 pass, `test_knowledge_indexer.py` 55/55 pass.

### #46 — CSP `unsafe-inline` (MEDIUM)
**Location:** `tools/app/server.py:722-730`.
**Root cause:** Global `unsafe-inline` for script + style across every response. Dashboard needs inline scripts; other endpoints did not.
**Fix:** Scoped `unsafe-inline` to `/dashboard` path only. All other paths get strict `default-src 'none'; script-src 'self'; style-src 'self'; ... object-src 'none'; base-uri 'self'`.
**Verify:** Imports clean.

### #47 — 401/403 auth oracle (MEDIUM)
**Location:** `tools/app/server.py:_verify_bearer_secret`, `verify_paperclip_secret`; `tools/app/paperclip_bridge.py:_verify_bearer`.
**Root cause:** Returned 401 for missing header but 403 for wrong bearer, plus distinct detail strings. Attacker could distinguish header-missing from secret-wrong, narrowing brute-force.
**Fix:** Single 401 "Unauthorized" for both cases. Constant-time compare runs unconditionally (empty string when header missing) — eliminates timing oracle too.
**Verify:** All three sites updated; `test_paperclip_bridge.py` + `test_admin_routes.py` + `test_server.py` pass.

### #51 — Double `mark_failed` race (MEDIUM)
**Location:** `tools/core/pipeline_state.py:570+`.
**Root cause:** `mark_failed` could be called concurrently by the async error handler + `nightly_catch_all` sweep. Second call ran a FAILED→FAILED transition, overwrote `error` field, and raced `save_state`.
**Fix:** Idempotent mark_failed — when state is already FAILED, append the new error to the 20-slot history buffer and skip the transition. Original root cause stays in `state.error`.
**Verify:** `test_pipeline_state.py` 73/73, `test_pipeline_retry.py` 45/45.

### #52 — Test pollution (HIGH, **MITIGATED**)
**Location:** `tools/tests/test_admin_routes.py:23`, `test_admin_routes_hardening.py`, `test_healthz_endpoint.py`, `test_metrics_endpoint.py`.
**Root cause:** Module-level `sys.modules.pop(...)` permanently removed conftest stubs for fastapi/slowapi/httpx/pydantic. Subsequent tests expecting stubs got real implementations and broke.
**Fix (this session):**
1. **`scripts/run_tests_isolated.sh`** — per-file runner that gives each test module its own process. Pollution cannot cross a process boundary. Recommended for CI verification.
2. **`ConfigDict` stub** added to conftest so transitive imports of `tools.app.server` work in stubbed environments.
**Deferred:** In-process containment (proper restore loop in `_reset_module_caches`) — attempted but produced worse cross-file pollution than the symptom. Full architectural fix requires refactoring the four "needs-real-modules" test files to restore at module-teardown.
**Verify:** Per-file run on every key module 100% pass (387 tests across 7 files).

## Architectural Decisions & Justifications

1. **CSP per-path scoping (#46)** — refused to remove `unsafe-inline` globally because `/dashboard` renders inline-script chart code. Per-path CSP gives strict default to webhooks/API and keeps the dashboard functional.

2. **Auth oracle fix returns 401 not 403 (#47)** — RFC 7235 says 401 = "auth needed or failed" and 403 = "auth succeeded but forbidden". For bearer-token webhooks the failure is always "auth failed", so 401 is RFC-correct. Removing the 403 path is also the GDPR/security minimum.

3. **Drive trash, not hard-delete on reset (#45)** — destructive ops on shared docs need a 30-day recovery window. Trash is reversible, hard-delete is not. Cost is identical (Drive quota counts trash).

4. **Retention default 90 days (#44)** — chat-archive industry standard; aligned with the "active cohort" lifecycle (15 lectures × 2 cohorts × roughly weekly = 30+ active days, plus buffer). Operator can override via `MESSAGE_RETENTION_DAYS`.

5. **Encryption deferred (#44)** — implementing AES-256-GCM here without an established key-management policy (where the key lives, how it rotates, how dev/staging/prod differ) would create more risk than it solves. The retention + erasure path is the minimum-viable GDPR posture; encryption is a separate Q3 deliverable.

6. **#52 in-process containment reverted** — tried it, made things worse. Per-process boundary (the isolated runner) is the only fix that holds.
