# ხარისხის კრიტიკული ანალიზი — მხოლოდ ინსტრუქტორისთვის

## 1. სწავლების ხარისხი — სიცხადე და ბუნდოვანი/არასრული საკითხები

**ძლიერი მხარეები:**
*   გულწრფელი ენთუზიაზმი და ღრმა პირადი გამოცდილება ყველა ინსტრუმენტთან მიმართებაში.
*   რეალურ დროში დემონსტრირება ქმნის ავთენტურ სასწავლო მომენტებს.
*   მეტაფორების კარგი გამოყენება (MCP როგორც "ჰაბი", Claude როგორც "მრჩეველთა საბჭო").
*   გულწრფელობა მოდელების შეზღუდვებსა და პირად პრეფერენციებზე.
*   კარგი კომუნიკაცია სტუდენტებთან; კითხვების დასმის წახალისება.

**პრობლემები სიცხადესთან დაკავშირებით:**

**კრიტიკული ბუნდოვანება MCP-ის შესახებ:** MCP სერვერების ახსნა არასრულია. სტუდენტებმა მოისმინეს, რომ "ეს არის ჰაბი, რომელსაც აპლიკაციებს უკავშირებთ", მაგრამ არ მიუღიათ კონცეპტუალური ახსნა, თუ რა არის რეალურად MCP (Model Context Protocol), როგორ მუშაობს ის არქიტექტურულად ან რატომ არის მნიშვნელოვანი. ეს არის ფუნდამენტური კონცეფცია, რომელსაც სჭირდებოდა 5-წუთიანი სტრუქტურირებული ახსნა და არა მხოლოდ ანალოგია და ლაივ-დემონსტრაცია, რომელიც ნაწილობრივ წარუმატებელი აღმოჩნდა.

**აგენტების დონეების (Tier) სისტემა ნახსენები იყო, მაგრამ არა მკაფიოდ განმარტებული:** სამდონიანი იერარქია (VS Code > Claude Desktop > Web interface) ლექციის ბოლოსკენ სწრაფად იქნა ნახსენები, მკაფიო ვიზუალური ჩარჩოს ან წერილობითი შეჯამების გარეშე. სტუდენტები, სავარაუდოდ, ვერ შეძლებდნენ მის გახსენებას ან დონეების ერთმანეთისგან გარჩევას ლექციის შემდეგ.

**"სუპერ აგენტის" აღწერა არ იყო დასაბუთებული:** ინსტრუქტორმა ახსენა 9-მოდელიანი სინთეზური სუპერ აგენტის შექმნა, მაგრამ ახსნა იყო ფრაგმენტული და წრიული. სტუდენტებმა ისე დატოვეს ლექცია, რომ ვერ გაიგეს რა არის OpenCanvas/Open Close, როგორია რეალური არქიტექტურა ან როგორ შეიძლება მისი გამეორება.

**უსაფრთხოების საკითხი Prompt Injection-ის შესახებ იყო საგანგაშო, მაგრამ დაუმუშავებელი:** ინსტრუქტორმა წამოჭრა სერიოზული საკითხი იმის შესახებ, რომ კონფიდენციალური მონა
როგორ შეიძლება მისი გამეორება.

**უსაფრთხოების საკითხი Prompt Injection-ის შესახებ იყო საგანგაშო, მაგრამ დაუმუშავებელი:** ინსტრუქტორმა წამოჭრა სერიოზული საკითხი იმის შესახებ, რომ კონფიდენციალური მონაცემები შეიძლება მოპოვებულ იქნას prompt injection-ის (პრომპტის ინექციის) საშუალებით — შემდეგ კი სხვა თემაზე გადავიდა ისე, რომ არ აუხსნია, რეალურად რა არის prompt injection, როგორ დავიცვათ თავი მისგან, ან რამდენად სერიოზულია რეალური რისკი.

**Computer Use-ის (კომპიუტერის გამოყენების) ახსნა იყო არათანმიმდევრული:** ინსტრუქტორმა აღნიშნა, რომ ChatGPT-ის Computer Use "სუსტია", მაგრამ არ მოუყვანია შედარებითი ბენჩმარკები ან მარცხის კონკრეტული მაგალითები. სტუდენტებმა არ იციან, ბრაუზერის ავტომატიზაციისთვის როდის რომელი ინსტრუმენტი გამოიყენონ.

## 2. კრიტიკული ხარვეზები — გამოტოვებული ან არასაკმარისად განხილული მნიშვნელოვანი თემები

**ხარვეზი 1: რა არის MCP?** Model Context Protocol-ის რეალური განმარტება არ ყოფილა მოცემული. სტუდენტები ვერ აუხსნიან კოლეგებს, რა არის ის, რატომ შექმნა Anthropic-მა, ან რით განსხვავდება ის API კავშირებისგან.

**ხარვეზი 2: ხარჯების მართვა** — მოკლედ იყო ნახსენები (100$-იანი პაკეტის დახარჯვა, Claude-ის ფასები), მაგრამ არ ყოფილა ფასების სისტემატური მიმოხილვა 5-ვე მოდელისთვის, API ხარჯების შედარება გამოწერის ხარჯებთან, ან როგორ უნდა დაიგეგმოს ბიუჯეტი AI ინსტრუმენტებისთვის. ქართული ბიზნესებისთვის ეს კრიტიკულად მნიშვნელოვანია.

**ხარვეზი 3: მონაცემთა კონფიდენციალურობა და შესაბამისობა** — ირიბად იქნა ნახსენები (Data Control-ის გათიშვა), მაგრამ არ განხილულა: GDPR-ის გავლენა, რა ემართება აშშ-ის სერვერებზე ატვირთულ ქართული ბიზნესის მონაცემებს, რომელ მოდელებს აქვთ ევროპული მონაცემთა დამუშავების შეთანხმებები. მენეჯერებისთვის, რომლებიც სენსიტიურ მონაცემებთან მუშაობენ, ეს უალტერნატივო საკითხია.

**ხარვეზი 4: როდის არ უნდა გამოვიყენოთ AI** — მთელი სესია სავსე იყო ენთუზიაზმით AI-ის შესაძლებლობების მიმართ. არ ყოფილა ნახსენები შემთხვევები, როდესაც AI უშვებს შეცდომებს, აქვს ჰალუცინაციები, რაც მნიშვნელოვან გავლენას ახდენს, ან ქმნის იურიდიულ/ბიზნეს რისკებს.

**ხარვეზი 5: ფა
მით AI-ის შესაძლებლობების მიმართ. არ ყოფილა ნახსენები შემთხვევები, როდესაც AI უშვებს შეცდომებს, აქვს ჰალუცინაციები, რაც მნიშვნელოვან გავლენას ახდენს, ან ქმნის იურიდიულ/ბიზნეს რისკებს.

**ხარვეზი 5: ფაილებთან და დოკუმენტებთან მუშაობა** — მოკლედ იყო ნახსენები Custom GPT-ებში ფაილების ატვირთვა, მაგრამ არასდროს ყოფილა დემონსტრირებული ან ახსნილი კომპანიის დოკუმენტებთან (PDF, Excel, Word) მუშაობის პრაქტიკული პროცესი სხვადასხვა მოდელში.

**ხარვეზი 6: Prompt Engineering-ის საფუძვლები** — Custom Instructions ნაჩვენები იყო, მაგრამ ლექციაზე "პრომპტინგი" მოხსენიებული იყო, როგორც სამომავლო თემა. იმის გათვალისწინებით, რომ ეს უკვე მესამე ლექციაა, პრომპტინგის საფუძვლები უკვე უნდა ყოფილიყო წარმოდგენილი.

**ხარვეზი 7: Claude Code ნახსენები იყო, მაგრამ არ უსწავლებიათ** — მიშას პროექტი გამოყენებული იყო დემონსტრაციისთვის, მაგრამ ის სტუდენტები, რომლებსაც ჯერ არ დაუყენებიათ Claude Code, პროცესს ჩამორჩნენ. დამწყებთათვის არ ყოფილა შეთავაზებული VS Code + Claude Code-ის ინსტალაციისა და ადაპტაციის გზა.

**ხარვეზი 8: Booking.com/სასტუმროს ექსპერიმენტი ფილოსოფიურად დაუსრულებელი დარჩა** — ინსტრუქტორსა და ტარიელს ჰქონდათ ხანგრძლივი დისკუსია იმის შესახებ, შეუძლია თუ არა LLM-ებისთვის 1000-ჯერ მიმართვას გავლენა მოახდინოს მოდელის პასუხებზე. ეს მომხიბლავი თემაა, მაგრამ გადაუჭრელი დარჩა. ინსტრუქტორს არ განუმარტავს, რეალურად რა ახდენს გავლენას LLM-ის რეკომენდაციებზე (სატრენინგო მონაცემების განახლების თარიღები, სტრუქტურირებული მონაცემები, ციტირებები, ვებ-ქროულინგი).

## 3. ტექნიკური სიზუსტე — უზუსტობები ან მოძველებული ინფორმაცია

**მტკიცება: "Claude ლიდერობს ყველა ბენჩმარკში გამონაკლისის გარეშე"** — ოდნავ გადაჭარბებულია. 2025 წლის მდგომარეობით, Claude 3.5/3.7 Sonnet ლიდერობს კოდირების ბენჩმარკებში (SWE-bench) და ლოგიკურ მსჯელობაში (reasoning), მაგრამ GPT-4o და Gemini Ultra ლიდერობენ გარკვეულ მულტიმოდალურ ბენჩმარკებში. ამ აბსოლუტურმა განცხადებამ შეიძლება შეცდომაში შეიყვანოს სტუდენტები და უგულებელყოს სხვა მოდელების ვალიდური გამოყენების შემთხვევები.

**მტკიცება Prompt Injection-ისა და სატრენინგო მონაცემების
მარკებში. ამ აბსოლუტურმა განცხადებამ შეიძლება შეცდომაში შეიყვანოს სტუდენტები და უგულებელყოს სხვა მოდელების ვალიდური გამოყენების შემთხვევები.

**მტკიცება Prompt Injection-ისა და სატრენინგო მონაცემების გაჟონვის შესახებ** — ინსტრუქტორმა მიანიშნა, რომ თუ მომხმარებელს ჩართული აქვს "Improve the Model" (მოდელის გაუმჯობესება), მის კონფიდენციალურ მონაცემებზე წვდომა შეიძლება მოიპოვოს სხვა მომხმარებელმა prompt injection-ის საშუალებით. ეს კონცეპტუალურად არეული ახსნაა. Prompt injection თავს ესხმის მოდელის ინსტრუქციების შესრულების უნარს და არა შენახულ სატრენინგო მონაცემებს. ის, რაც ინსტრუქტორმა აღწერა, უფრო ზუსტად არის RAG სისტემების ან fine-tuning (დაზუსტების) პროცესების პრობლემა და არა სტანდარტული ჩატ-ინტერფეისების. ეს საჭიროებდა შესწორებას და არა ირიბ მუქარად დატოვებას.

**მტკიცება: "Perplexity მეორე ადგილზეა კონექტორებში"** — სადავოა. 2025 წლის შუა პერიოდისთვის, ChatGPT-ის კონექტორების ეკოსისტემა Zapier-თან და ნატიურ ინტეგრაციებთან ერთად, სავარაუდოდ, უფრო განვითარებულია, ვიდრე Perplexity-ის.

## ნაწილი I — სწავლების ხარისხის შეჯამება (პუნქტები 1–6)

**1. სიცხადე (Clarity):**
გამორჩეული პრაქტიკული ცოდნა, რომელსაც ფარავს ინფორმაციის დაბალი სიმჭიდროვე. ლექტორის ექსპერტიზა ნამდვილი და თვალსაჩინოა, მაგრამ სიტყვიერი გამეორებები და ყოყმანი მნიშვნელოვნად ამცირებს "სიგნალი-ხმაურის" (signal-to-noise) თანაფარდობას. ერთი და იგივე აზრი ხშირად სამჯერ მეორდება, მაშინ როცა ერთხელ — მტკიცებულებით გამყარებული — გაცილებით ეფექტური იქნებოდა.

**2. კრიტიკული ხარვეზები (Gaps):**
ამ ლექციის "ჭერს" ოთხი კრიტიკული დანაკლისი განსაზღვრავს: არ არის ფასწარმოქმნის/ღირებულების გზამკვლევი, არ არის პრაქტიკული სავარჯიშოები მონაწილეებისთვის, არ არის შედეგების შეფასების ჩარჩო და არ განიხილება ტოკენების/კონტექსტის შეზღუდვები. ეს არ არის მცირე გამოტოვებები — ეს არის ნაპრალი AI-ის *მაყურებელ* და AI-ის *მომხმარებელ* მონაწილეებს შორის.

**3. ტექნიკური სიზუსტე (Accuracy):**
მრავალრიცხოვანი გადაჭარბებული განცხადებები AI ლანდშაფტის დამახინჯებულ სურათს ქმნის. ყველაზე დამაზიანებელია Claude-ის წარმოჩენა, როგორც უპირობო ლიდერისა ყველა ბენჩმარკში — ეს 2025 წლისთვის ფაქტობრივად სადავოა და მონაწილეებს არასწორ პრეზუმფციებზე დაყრდნობით გადაწყვეტილებების მიღებას ასწავლის. ინსტრუმენტების მიმართ პირადი სიმპათიები ობიექტურ შეფასებებადაა გასაღებული.

**4. პედაგოგიკა (Pedagogy):**
სესია სტრუქტურირებულია როგორც გახანგრძლივებული დემონსტრაცია და არა როგორც სასწავლო პროცესი. ლექტორის ეკრანის დროისა და მონაწილეთა ჩართულობის დროის თანაფარდობა დაახლოებით 95:5-ზეა. ეს კურსის ყველაზე გადაუდებელი სტრუქტურული პრობლემაა.

**5. ტემპი (Pacing):**
ლექცია ცდილობს დაფაროს მასალა, რომელიც სამ სესიას უნდა ეთმობოდეს. ეს სისტემური პრობლემაა და არა ერთჯერადი შემთხვევა. თითოეული თემა მხოლოდ

**6. სტრუქტურა (Structure):**

### 10. კრიტიკული ბრმა წერტილები — გადამწყვეტი AI კონცეფციები 2025–2026 წლებისთვის
