### 2. ძირითადი კონცეფციები

* **ინფრასტრუქტურაზე ორიენტირებული მიდგომა:** არასოდეს დაიწყოთ პროექტი Claude Code-ისთვის შემთხვევითი დავალებების მიცემით; ჯერ მოაწყვეთ ფაილების სწორი სტრუქტურა, უსაფრთხოება და Git ინტეგრაცია.
* **უნარები (Skills), როგორც მუდმივი კონტექსტი:** უნარები არის ინსტრუქციების მრავალჯერადი გამოყენების ნაკრები, რომლებსაც Claude ავტომატურად იძახებს შესაბამისი დავალებების მიცემისას — მათი ხელით გამოძახება საჭირო არ არის.
* **ღია კოდი (Open source) vs. დახურული კოდი:** საჯარო GitHub რეპოზიტორიები საზოგადოებას აძლევს შემოწმებისა და კონტრიბუციის საშუალებას; პირადი (private) რეპოზიტორიები გამოიყენება პერსონალური/ბიზნეს პროექტებისთვის.
* **Commit და Push სამუშაო პროცესი:** ლოკალური შენახვა (commits) + ქლაუდ ბექაფი (GitHub push) პროექტის უსაფრთხოებისთვის.
* **NotebookLM-ისა და Claude-ის ურთიერთშემავსებლობა:** NotebookLM საუკეთესოა წყაროებზე დაფუძნებული ანალიზისა და თვითსწავლისთვის; Claude საუკეთესოა სტრუქტურირებული შედეგებისა და გარე გამოყენების მასალების (deliverables) შესაქმნელად.

### 3. პრაქტიკული მაგალითები და დემონსტრაციები

1. VS Code-ის ახალი პროექტის ნულიდან შექმნა და Claude Code-ის გახსნა.
2. Claude Code-ისთვის დეტალური პრომპტის დაწერა airmpi.com-დან უნარების/ინსტრუმენტების ინტეგრირებისთვის.
3. Claude Code-ის გამოყენება Draw.io-ში ორგანიზაციული სტრუქტურის შესაქმნელად (Dark mode, 500-თანამშრომლიანი კომპანია).
4. "Travel Price Monitor" (მოგზაურობის ფასების მონიტორინგი) უნარის შექმნა ინტერაქტიული კითხვა-პასუხით Claude Code-ში.
5. Excel-ში Claude-ის Add-in-ის ჩვენება.
6. NotebookLM-ში ნავიგაცია ქართული ლიტერატურის წყაროებით ("ცისფერყანწელები").
7. გონებრივი რუკების (mind maps), ქვიზების, ფლეშქარდე

## ძირითადი თემები

- AI კოდირების პროექტებისთვის ყოველთვის შექმენით სწორი ინფრასტრუქტურა Claude Code-ის გამოყენებამდე.
- ლექტორის GitHub სეთაფის რეპოზიტორია (`/setup` ბრძანება) პროექტის მყარი საძირკვლისთვის უნდა იქნას გამოყენებული.
- Claude-ის უნარები (Skills) არის ძლიერი, მრავალჯერადი გამოყენების ინსტრუქციების ნაკრები განმეორებადი დავალებებისთვის.
- NotebookLM საუკეთესოა წყაროებზე დაფუძნებული ანალიზისა და თვითსწავლისთვის, ხოლო Claude — სტრუქტურირებული ბიზნეს შედეგების შესაქმნელად.
- უსაფრთხოების პრაქტიკა, მათ შორის VS Code-ის ნებართვების ხელით დადასტურება, გადამწყვეტია.
- Git და GitHub უზრუნველყოფენ პროექტის ვერსიების კონტროლს, ბექაფს და გუნდურ თანამშრომლობას.

### ინსტრუმენტები
- [[Claude Code]] -- ეს არის AI ინტეგრირებული კოდის რედაქტორი, რომელიც ეხმარება დეველოპერებსა და არადეველოპერებს პროექტების შექმნასა და მართვ
- [[Git]] -- ეს არის ლოკალური ვერსიების კონტროლის სისტემა, რომელიც საშუალებას აძლევს მომხმარებლებს აკონტროლონ ცვლილებები ფაილებში და 
- [[CLI-Anything]] -- ეს არის GitHub რეპოზიტორიიდან გადმოსაწერი ინსტრუმენტი, რომელიც Claude Code-ს საშუალებას აძლევს დაუკავშირდეს და მართოს ლო

### პლატფორმები
- [[GitHub]] -- ღრუბელზე დაფუძნებული პლატფორმა, რომელიც მასპინძლობს Git რეპოზიტორიებს, რაც საშუალებას იძლევა კოდის ვერსიების მართვის, სა
- [[NotebookLM]] -- Google-ის AI პლატფორმა, რომელიც განკუთვნილია დოკუმენტების დამუშავების, ანალიზისა და კონტენტის გენერირებისთვის, წყაროებზე
