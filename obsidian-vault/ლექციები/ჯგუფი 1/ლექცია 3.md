# AI აგენტები და აგენტური ხელოვნური ინტელექტი

> ჯგუფი #1 -- ლექცია #3

## შეჯამება

როგორც AI ტრენინგების ექსპერტი ანალიტიკოსი, წარმოგიდგენთ ლექციის ტრანსკრიპტის დეტალურ და სრულყოფილ შეჯამებას ქართულ ენაზე. 

ეს ლექცია (დაახლოებით 1.5 საათიანი ძირითადი მასალა) წარმოადგენდა კურსის მე-4 მოდულზე წინსწრებით გადასვლას და ფოკუსირებული იყო **AI აგენტებსა და აგენტურ ხელოვნურ ინტელექტზე (Agentic AI)**. 

ქვემოთ მოცემულია ლექციის სტრუქტურირებული ანალიზი:

### 1. მთავარი თემები (რა განიხილეს ლექციაზე)
ლექციაზე განხილული იყო შემდეგი საკვანძო საკითხები:
* **AI აგენტები ავტომატიზაციის წინააღმდეგ** — ფუნდამენტური კონცეპტუალური სხვაობა ამ ორ ტერმინს შორის.
* **AI აგენტების განვითარების პლატფორმები** — n8n, Zapier, Make.com და Claude Code-ზე გადასვლის ტენდენცია.
* **Claude Desktop აპლიკაცია** — მისი სამი ძირითადი რეჟიმი: Chat, Cowork და Code.
* **VS Code როგორც IDE (ინტეგრირებული განვითარების გარემო)** — პროგრამის დაყენება, ექსტენსიები და Claude Code-ის ინტეგრაცია.
* **MCP (Model Context Protocol)** — პროტოკოლი AI-ის გარე სერვისებთან დასაკავშირებლად.
* **CLI (Command Line Interface) ინსტრუმენტები** — Playwright CLI და Anything CLI კომპიუტერის მართვისა და ვებ-სკრეპინგისთვის.
* **ქლაუდ (Cloud) სერვისების მიმოხილვა** — Google Cloud, Microsoft Azure და AWS (Amazon).
* **Google Antigravity IDE** — VS Code-ის ალტერნატივა Gemini-ს ინტეგრაციით.
* **პრაქტიკული HR აგენტის დემონსტრაცია** — AI-ზე დაფუძნებული რეკრუტინგის ასისტენტის ნულიდან შექმნა.

### 2. ძირითადი კონცეფციები (ახალი ცნებები და იდეები)

* **AI აგენტი ავტომატიზაციის წინააღმდეგ:** ლექტორმა მკაფიოდ გამიჯნა ეს ორი ცნება. ავტომატიზაცია მიჰყვება ფიქსირებულ, ხაზოვან ბრძანებებს (მაგ. "ყოველ სამშაბათს 8:00 საათზე ჩართე Zoom"). AI აგენტი კი იღებს ზოგად ინსტრუქციებსა და ინსტრუმენტებს, რის შემდეგაც *დამოუკიდებლად იღებს გადაწყვეტილებას*, თუ რა მოქმედება შეასრულოს, როდის და როგორ. მას აქვს გადაწყვეტილების მიმღები შრე შემომავალ და გამავალ ინფორმაციას შორის.
* **პარადიგმის ცვლილება (2026 წლის იანვარი):** ადრე n8n დომინირებდა აგენტური პროცესების ასაწყობად ვიზუალური "კვანძების" (nodes) გამოყენებით. 2026 წლის იანვრიდან C
მომავალ და გამავალ ინფორმაციას შორის.
* **პარადიგმის ცვლილება (2026 წლის იანვარი):** ადრე n8n დომინირებდა აგენტური პროცესების ასაწყობად ვიზუალური "კვანძების" (nodes) გამოყენებით. 2026 წლის იანვრიდან Claude Code იმდენად გაძლიერდა კოდის წერაში, რომ ახლა შეუძლია: 1. აგენტური პროცესების ნულიდან, კოდით შექმნა n8n-ის გარეშე; 2. n8n-თან MCP-ით დაკავშირება და იქ პროცესების პროგრამულად აწყობა; 3. ჰიბრიდული მიდგომის გამოყენება.
* **n8n არქიტექტურა:** იყენებს წინასწარ გამზადებულ კვანძებს (რომლებიც შეიცავს ფიქსირებულ კოდს). მომხმარებლები მათ ვიზუალურად აკავშირებენ. მისი ძლიერი მხარე საიმედოობაა, თუმცა შეზღუდულია სირთულითა და მოუქნელობით.
* **MCP (Model Context Protocol):** ახსნილია, როგორც "დამაკავშირებელი ხიდი" — უნივერსალური პროტოკოლი, რომელიც Claude Code-ს აკავშირებს გარე სერვისებთან (მონაცემთა ბაზები, Slack, Google Sheets, Gmail). ლექტორმა ის USB კაბელს შეადარა.
* **CLI (Command Line Interface):** კიდევ ერთი დამაკავშირებელი ინსტრუმენტი, რომელსაც MCP-ზე მეტი მოქმედების შესრულება შეუძლია. მაგალითად, Playwright-ისთვის CLI უფრო სწრაფი, იაფი და ეფექტურია, ვიდრე MCP.
* **VS Code:** აღწერილია, როგორც პროგრამისტის სამუშაო მაგიდა (IDE), რომელიც აერთიანებს კოდის რედაქტორს, ტერმინალს და ფაილების მენეჯერს. მეტაფორა: "სამზარეულო, სადაც დანა, ქვაბი, ღუმელი და მაცივარი ერთ სივრცეშია".
* **Claude Desktop-ის 3 რეჟიმი:** 
  1. *Chat* - სტანდარტული სასაუბრო ინტერფეისი.
  2. *Cowork* - გამარტივებული ვერსია არატექნიკური მომხმარებლებისთვის, მუშაობს ლოკალურ ფაილებთან და ინტერნეტთან.
  3. *Code* - სრული შესაძლებლობები, უკავშირდება ქლაუდს და გარე აპლიკაციებს (მოითხოვს მეტ ტექნიკურ ცოდნას).
* **უნარები (Skills) და პლაგინები:** სპეციალიზებული უნარები (Frontend, SEO, Excel Analytics და ა.შ.), რომლებიც Claude Code-ს ემატება და ზრდის მის ეფექტურობას კონკრეტულ ამოცანებში ("როგორც ოსტატისთვის შესაბამისი ინსტრუმენტების მიცემა").

### 3. პრაქტიკული მაგალითები და დემონსტრაციები

* **დემო 1: n8n Workflow ვიზუალიზაცია:** ლექტორმა აჩვენა n8n.cloud-ის ინტერფეისი რთული სქემით (AI Agent, OpenAI, Pinecone Vector Store). აჩვენა AI აგენტის კვანძი დაკავშირებული ინსტრუმენტებით (კალკულატორი, Slack, Google Sheets, Gmail).
* **დემო 2: Claude Cowork — HR დავალება:** Claude-ის Cowork რეჟიმში შეყვანილ იქნა რთული პრომპტი: საქართველოს შრომის ბაზრის კვლევა (2026 წლის მარტი), ხელფასების დადგენა, IT მენეჯერის ვაკანსიის 3 ვარიანტის შექმნა Word-ში, Playwright CLI-ით LinkedIn-ზე ტოპ 10 კანდიდატის მოძიება, PowerPoint პრეზენტაციის შექმნა და Excel-ში შედარებითი ანალიზის გაკეთება. AI-მ დასვა დამაზუსტებელი კითხვები (კომპანიის ზომა, ენა და ა.შ.).
* **დემო 3: VS Code + Claude Code — HR აგენტის სრული აწყობა:**
  ნაბიჯ-ნაბიჯ პროცესი VS Code-ში სპეციალური ბრძანებებით:
  - `/init-project` — AI-მ შექმნა ფოლდერების სტრუქტურა.
  - `/create-prd` — AI-მ დააგენერირა პროდუქტის მოთხოვნების დოკუმენტი.
  - `/architecture-review` — AI-მ შექმნა არქიტექტურა (React, Node.js, PostgreSQL, Claude 3 + Gemini).
  - `/generate-tests` და `/implement` — AI-მ დაწერა ტესტები და რეალური კოდი.
  - `/commit` — კოდი აიტვირთა GitHub-ზე.
  *აგენტის ფუნქციები მოიცავდა:* Gmail-ის მონიტორინგს, Telegram ნოტიფიკაციებს, ვექტორულ ბაზას, Enagram AI-ით ინტერვიუს ტრანსკრიფციას და LinkedIn-დან კანდიდატების პოსტების სკრეპინგს.
* **დემო 4: Draw.io ინფოგრაფიკა:** Cowork რეჟიმში, Anything CLI-ის გამოყენებით, ერთი წინადადებით შეიქმნა Claude Code-ის ეკოსისტემის ინფოგრაფიკა Draw.io-ში.
* **დემო 5: Google Antigravity IDE:** ლექტორმა სცადა Google-ის ახალი IDE-ის ჩვენება. მიუხედავად იმისა, რომ პროგრამა მეხსიერების პრობლემის გამო გაითიშა, ლექტორმა აჩვენა წინა სესიაზე Gemini-ს მიერ შექმნილი ქართული სამზარეულოს ვებსაიტი (AI გენერირებული სურათებით).

### 5. მოქმედების ნაბიჯები (რა უნდა გააკეთონ მონაწილეებმა)

ლექტორის მიერ გაცემული დავალებები და რეკომენდაციები:
1. **გადმოწერეთ და დააინსტალირეთ VS Code** (უფასოდ code.visualstudio.com-დან).
2. **დააყენეთ VS Code-ის ორი ექსტენსია:** Claude (ნარინჯისფერი აიქონით) და Codex/ChatGPT.
3. **გამოიწერეთ Claude** (მინიმუმ $20/თვეში) და დააკავშირეთ VS Code-თან.
4. **გადმოწერეთ Claude Desktop App** (Mac ან Windows-ისთვის).
5. **გადმოწერეთ Google Antigravity IDE.**
6. **დარეგისტრირდით n8n.io-ზე** (სამომავლო გამოყენებისთვის).

## ძირითადი თემები

- AI აგენტების ინდუსტრია ფუნდამენტურად შეიცვალა (2026 წლის იანვრიდან) — Claude Code-ის შესაძლებლობები უკვე იძლევა რთული აგენტების შექმნის საშუალებას ტრადიციული პლატფორმების (როგორიცაა n8n) გარეშე.

- [[Embeddings]] (ემბედინგი) -- ტექსტის, სურათების ან სხვა მონაცემების რიცხვითი წარმოდგენა ვექტორულ სივრცეში, რომელიც აღბეჭდავს მათ სემანტიკურ მნიშვნელო
- [[AI Safety]] (AI უსაფრთხოება) -- პრინციპები და მეთოდები, რომლებიც უზრუნველყოფს AI სისტემების უსაფრთხო, საიმედო და ეთიკურ ფუნქციონირებას.
- [[Compliance]] (შესაბამისობა) -- ბიზნესის ვალდებულება, შეასრულოს შესაბამისი კანონები, რეგულაციები, სტანდარტები და შიდა პოლიტიკა, განსაკუთრებით AI სისტემე
- [[Human-in-the-loop]] (ადამიანი-მარყუჟში) -- AI სისტემების დიზაინი, სადაც ადამიანის ჩარევა და ზედამხედველობა ინარჩუნებს კონტროლს კრიტიკულ გადაწყვეტილებებსა და პროცეს
- [[AI Governance]] (AI მმართველობა) -- პოლიტიკის, პროცესებისა და პასუხისმგებლობების სისტემა AI სისტემების დიზაინის, განვითარების, დანერგვისა და გამოყენების ზედ
- [[EU AI Act]] (ევროკავშირის AI აქტი) -- ევროკავშირის რეგულაცია, რომელიც ადგენს AI სისტემების გამოყენების სამართლებრივ ჩარჩოს, კლასიფიცირებს რისკებს და მოითხოვს 
- [[Hallucinations (AI)]] (ჰალუცინაციები (AI)) -- AI მოდელების ტენდენცია, შექმნან არასწორი ან არარსებული ინფორმაცია, რომელიც დამაჯერებლად ჟღერს.
- [[Data Pipeline Architecture]] (მონაცემთა ნაკადის არქიტექტურა) -- სისტემის დიზაინი, რომელიც აღწერს, თუ როგორ მიედინება მონაცემები სხვადასხვა წყაროდან, როგორ მუშავდება, ტრანსფორმირდება და
- [[Voice AI]] (ხმოვანი AI) -- ხელოვნური ინტელექტი, რომელიც ამუშავებს და რეაგირებს ხმოვან ინფორმაციაზე, მაგალითად, მეტყველების ტრანსკრიფციისთვის.
- [[Multimodal Agents]] (მულტიმოდალური აგენტები) -- AI აგენტები, რომლებსაც შეუძლიათ მრავალი ტიპის მონაცემების (მაგ. ტექსტი, ხმა, სურათი) აღქმა, დამუშავება და მათზე რეაგირებ
- [[AI Agent Memory]] (AI აგენტის მეხსიერება) -- მექანიზმები, რომლებიც საშუალებას აძლევს AI აგენტებს შეინარჩუნონ კონტექსტი და ინფორმაცია სხვადასხვა სესიებს შორის, ანუ ჰქ
- [[State Management]] (მდგომარეობის მართვა) -- პროცესი, რომელიც აკონტროლებს და ინახავს AI აგენტის მიმდინარე მდგომარეობას ან კონტექსტს, განსაკუთრებით საუბრისას.
- [[Local/Edge AI]] (ლოკალური/Edge AI) -- AI მოდელების გაშვება მოწყობილობაზე ან ლოკალურ ინფრასტრუქტურაზე, ქლაუდში გაგზავნის ნაცვლად, კონფიდენციალურობისა და სიჩქარ
- [[AI-based Analytics]] (AI-ზე დაფუძნებული ანალიტიკა) -- ხელოვნური ინტელექტის გამოყენება მონაცემების ანალიზისთვის, ტენდენციების იდენტიფიცირებისთვის და პროგნოზების გასაკეთებლად.
- [[Business Intelligence (BI)]] (ბიზნეს ინტელექტი (BI)) -- ტექნოლოგიები, სტრატეგიები და პროცესები, რომლებიც გამოიყენება ბიზნეს მონაცემების გასაანალიზებლად და მისი ღირებული ინფორმა
- [[ROI (Return on Investment)]] (ინვესტიციის უკუგება) -- საზომი, რომელიც აფასებს ინვესტიციის ეფექტურობას ან მომგებიანობას.
- [[API (Application Programming Interface)]] -- პროგრამული ინტერფეისი, რომელიც საშუალებას აძლევს ორ პროგრამულ კომპონენტს, დაუკავშირდნენ ერთმანეთს.
- [[UI (User Interface)]] -- მომხმარებლის ინტერფეისი, რომლის მეშვეობითაც მომხმარებელი ურთიერთქმედებს პროგრამულ უზრუნველყოფასთან ან აპარატურასთან.
- [[Frontend]] -- ვებ აპლიკაციის ის ნაწილი, რომელთანაც მომხმარებელი უშუალოდ ურთიერთქმედებს (ვიზუალური მხარე).
- [[Backend]] -- ვებ აპლიკაციის ის ნაწილი, რომელიც მართავს მონაცემებსა და სერვერულ ლოგიკას, მომხმარებლისთვის უხილავი.
- [[localhost]] -- კომპიუტერული ქსელის ჰოსტის სახელი, რომელიც მიუთითებს საკუთარ კომპიუტერზე.
- [[Workflow (n8n)]] -- n8n-ში ვიზუალურად აწყობილი პროცესების ნაკადი, რომელიც აერთიანებს სხვადასხვა კვანძებსა და სერვისებს.

### ტექნიკები

- [[Multi-agent Orchestration]] (მულტი-აგენტური ორკესტრირება) -- რამდენიმე AI აგენტის კოორდინაცია და მართვა, რათა ერთობლივად შეასრულონ რთული ამოცანები.
- [[Risk Classification]] (რისკების კლასიფიკაცია) -- AI სისტემებთან დაკავშირებული პოტენციური საფრთხეების იდენტიფიცირება, შეფასება და კატეგორიზაცია მათი ალბათობისა და ზემოქმე
- [[Evaluation of AI-generated Results]] (AI-ის გენერირებული შედეგების შეფასება) -- მეთოდები, რომლებიც გამოიყენება AI აგენტების მიერ მიღებული შედეგების სიზუსტის, ხარისხისა და ვალიდურობის შესამოწმებლად.
- [[Testing (AI Agents)]] (ტესტირება (AI აგენტები)) -- AI აგენტების ფუნქციონალურობის, საიმედოობის და უსაფრთხოების შემოწმების პროცესი.
- [[Quality Control (AI Agents)]] (ხარისხის კონტროლი (AI აგენტები)) -- პროცესები და მექანიზმები, რომლებიც უზრუნველყოფენ AI აგენტების მიერ შესრულებული ამოცანების მაღალ ხარისხს და სიზუსტეს.
- [[Error Management]] (შეცდომების მართვა) -- AI სისტემებში შეცდომების იდენტიფიცირების, ჩაწერის, ანალიზისა და გამოსწორების პროცესი.
- [[Logging]] (ლოგირება) -- სისტემის მოვლენების, აქტივობებისა და შეცდომების ჩაწერა მონიტორინგისა და დიაგნოსტიკისთვის.
- [[Monitoring]] (მონიტორინგი) -- AI სისტემების მუშაობის, რესურსების გამოყენებისა და ქცევის უწყვეტი თვალთვალის პროცესი.
- [[Chain-of-Thought]] -- პრომპტ-ინჟინერიის ტექნიკა, რომელიც AI-ს სთხოვს, გამოიტანოს შუალედური აზრები საბოლოო პასუხამდე მისვლამდე.
- [[Claude Code]] -- Anthropic-ის Claude AI მოდელის კოდის გენერირებისა და მართვის შესაძლებლობები, რომელიც იმდენად გაძლიერდა, რომ შეუძლია აგენ
