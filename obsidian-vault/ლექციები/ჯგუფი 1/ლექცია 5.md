### 1. მთავარი თემები (რა თემები განიხილეს ლექციაზე)

**ნაწილი 1: Claude-ის ეკოსისტემის განმარტება და VS Code პროექტის აწყობა (ლექციის დროის ~60%)**
*   **Claude-ის ეკოსისტემა:** განმარტება, თუ სად არსებობს Claude და რომელი ინტერფეისი უნდა გამოვიყენოთ კონკრეტულ შემთხვევაში (ვებსაიტი, Desktop აპლიკაცია თუ VS Code).
*   **პროექტის ინიციალიზაცია:** ნაბიჯ-ნაბიჯ სამუშაო პროცესი (workflow) VS Code-ში Claude Code-ის გამოყენებით.
*   **Git vs. GitHub:** მათ შორის არსებული ფუნდამენტური სხვაობის ახსნა.
*   **კავშირის მეთოდები:** სხვაობა API გასაღებებს (keys), ბრაუზერის/კომპიუტერის გამოყენებასა (Browser/Computer Use) და Playwright CLI-ს შორის.
*   **ტექნიკური მხარდაჭერა:** ინდივიდუალური დახმარება და პრობლემების მოგვარება მონაწილეებისთვის (ეკო, შორენა, მაკა, კესო/ქეთი).

**ნაწილი 2: ჩინური ხელოვნური ინტელექტის მოდელების მიმოხილვა (ლექციის დროის ~25%)**
*   **7 ჩინური AI მოდელის გაცნობა:** DeepSeek, Qwen, Kimi, Manus AI, Genspark AI, Skywork AI, MiniMax.
*   **შედარებითი ანალიზი:** ვებგვერდების გენერირების ლაივ-დემონსტრაციები Qwen, Kimi 2.5 და MiniMax მოდელების გამოყენებით.
*   **Manus AI:** პრეზენტაციების შექმნის უნიკალური შესაძლებლობების დემონსტრირება.

**ნაწილი 3: კითხვა-პასუხი (Q&A) და ინდივიდუალური მხარდაჭერა (ლექციის დროის ~15%)**
*   მონაწილეთა ვრცელი კითხვების განხილვა API კავშირების, Google AI Studio-სა და ბილინგის (გადახდების) შესახებ.
*   ეკრანის გაზიარებით პრობლემების მოგვარების ინდივიდუალური სესიები.

### 2. ძირითადი კონცეფციები (რა ახალი ცნებები და იდეები იქნა ახსნილი)

**Claude-ის ინტერფეისების იერარქია:**
*   **Claude ვებსაიტი (claude.ai):** აქვს საბაზისო ჩატის და კოდის ფუნქციები; Cowork ფუნქცია აქ *არ არის* ხელმისაწვდომი. ეს არის ყველაზე ნაკლებად რეკომენდებული ვარიანტი.
*   **Claude Desktop აპლიკაცია:** ვებსაიტზე ბევრად ეფექტურია; აქვს Cowork და Code ფუნქციები. რეკომენდებულია პროექტის საწყისი ბრეინშტორმინგისა დ
წვდომი. ეს არის ყველაზე ნაკლებად რეკომენდებული ვარიანტი.
*   **Claude Desktop აპლიკაცია:** ვებსაიტზე ბევრად ეფექტურია; აქვს Cowork და Code ფუნქციები. რეკომენდებულია პროექტის საწყისი ბრეინშტორმინგისა და კონსულტაციისთვის.
*   **VS Code და Claude Code:** ყველაზე მძლავრი ვარიანტი სრულმასშტაბიანი აგენტების/პროექტების ასაშენებლად. წარმოადგენს ძირითად სამუშაო სივრცეს დეველოპმენტისთვის.

**პროექტის განვითარების სამუშაო პროცესი (3 ნაბიჯი):**
1.  **ნაბიჯი 1 — საბაზისო ინფრასტრუქტურის მოწყობა:** გამოიყენეთ წინასწარ გამზადებული GitHub რეპოზიტორის შაბლონი (`claude-code-setup`) პროექტის ჩონჩხის შესაქმნელად. ჩასვით `settings.json` კონფიგურაცია Claude Code-ში და სთხოვეთ მას "დააინსტალიროს ეს გლობალურად" (install this globally). ეს ქმნის მრავალჯერადი გამოყენების პროექტის სტრუქტურას ყველა მომავალი პროექტისთვის. კეთდება მხოლოდ ერთხელ.
2.  **ნაბიჯი 2 — გაუშვით `/setup` ბრძანება:** ინფრასტრუქტურის ინსტალაციის შემდეგ, გახსენით ახალი სესია და გაუშვით `/setup` ბრძანება. ეს ავტომატურად აკონფიგურირებს პროექტის გარემოს. უკვე აწყობილი პროექტებისთვის `/setup`-ს შეუძლია არსებული კონფიგურაციის აუდიტი და გამოსწორების გზების შეთავაზება.
3.  **ნაბიჯი 3 — უნარების (Skills) და ხელსაწყოების (Tools) ინსტალაცია:** მიაწოდეთ Claude Code-ს უნარების პლატფორმის ბმული (aitmpl.com/skills) და დაავალეთ აირჩიოს და დააინსტალიროს საუკეთესო უნარები, აგენტები, ბრძანებები, ჰუკები (hooks), პარამეტრები და MCP-ები. სთხოვეთ მას ახსნას თითოეული ხელსაწყო, როდის უნდა გამოიძახოთ ისინი და რა სამომავლო დამატებები შეიძლება გახდეს საჭირო.

**პროექტის იდეაციის (Ideation) პროცესი:**
*   სანამ VS Code-ს შეეხებით, ინტენსიურად იბრეინშტორმინგეთ Claude-თან ჩატში (Desktop აპლიკაციაში).
*   აღწერეთ თქვენი პროექტი, სთხოვეთ რეკომენდაციები, დააზუსტეთ, რომ არ ხართ დეველოპერი.
*   *მაგალითი:* ერთ-ერთმა სტუდენტმა 2 სრული დღე დაუთმო მხოლოდ Claude-თან საუბარსა და დაგეგმვას, სანამ უშუალოდ შენებას დაიწყებდა.
*   მხოლოდ საფუძვლიანი დაგეგმვის შემდეგ გადადით VS Code-ში რეალური დეველოპმენტისთვის.

**Git vs. GitHub:**
*   **Git** = ლოკალური ვერსიების კონ
საუბარსა და დაგეგმვას, სანამ უშუალოდ შენებას დაიწყებდა.
*   მხოლოდ საფუძვლიანი დაგეგმვის შემდეგ გადადით VS Code-ში რეალური დეველოპმენტისთვის.

**Git vs. GitHub:**
*   **Git** = ლოკალური ვერსიების კონტროლი (წარმოიდგინეთ, როგორც ასლების შენახვა თქვენს პირად "სახელოსნოში").
*   **GitHub** = დისტანციური სარეზერვო რეპოზიტორი (წარმოიდგინეთ, როგორც ასლების გაგზავნა "ბანკის საცავში" უსაფრთხოებისთვის).

**API გასაღებები vs. ბრაუზერის/კომპიუტერის გამოყენება:**
*   **API Key** = პირდაპირი პროგრამული კავშირი სერვისებს შორის (მუდმივი, კოდიდან კოდში კომუნიკაცია).
*   **Computer Use / Playwright CLI** = შემოვლითი გზა (workaround), როდესაც API წვდომა არ არსებობს; ამ დროს AI ფიზიკურად ნავიგაციას უწევს ბრაუზერის ინტერფეისებს, როგორც ადამიანი.
*   **MCP** = პირდაპირი კავშირის კიდევ ერთი ფორმა (Model Context Protocol).

### 4. საკვანძო დასკვნები (ლექციის მთავარი დასკვნები და takeaways)

1.  **VS Code არის თქვენი მთავარი სამუშაო სივრცე:** შეწყვიტეთ Claude-ის ვებსაიტის გამოყენება; გადადით Desktop აპლიკაციაზე იდეების დასაგენერირებლად (ბრეინშტორმინგისთვის) და VS Code-ზე უშუალოდ პროექტის ასაშენებლად.
2.  **დაიცავით 3-ნაბიჯიანი წესი:** აუცილებლად მიჰყევით 3-ნაბიჯიან აწყობის პროცესს, სანამ Claude Code-ს რაიმე კონკრეტულ საპროექტო დავალებას მისცემთ.
3.  **ჩინური AI მოდელების ძალა:** ისინი წარმოადგენენ ძალიან ღირებულ ალტერნატივებს — განსაკუთრებით უფასო სურათების/ვიდეოების გენერირებისთვის (Qwen), პრეზენტაციებისთვის (Manus AI) და აგენტებში დაბალბიუჯეტიანი API ინტეგრაციისთვის (DeepSeek).
