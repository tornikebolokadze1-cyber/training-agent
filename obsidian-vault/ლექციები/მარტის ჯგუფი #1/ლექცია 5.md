# ჩინური AI მოდელების მიმოხილვა და Claude Code-ის პროექტის გამართვა

## შეჯამება

წარმოგიდგენთ ლექციის ტრანსკრიპტის დეტალურ შეჯამებასა და ექსპერტულ ანალიზს ქართულ ენაზე, მოთხოვნილი სტრუქტურის სრული დაცვით:

## ლექციის სრულყოფილი შეჯამება

### 1. მთავარი თემები
თორნიკე ბოლქვაძის ლექცია მოიცავდა ორ მთავარ სეგმენტს ქართველი მონაწილეების მცირე ჯგუფისთვის (შორენა, ნინო, ეკო, მაკა, კობა, კესო და სხვები):

**სეგმენტი A: Claude Code-ის პროექტის ინფრასტრუქტურის გამართვა (~პირველი 15 წუთი)**
* VS Code-ში Claude Code-ის პროექტის ჩონჩხის/შაბლონის შექმნა და გამართვა.
* `/setup` ბრძანების გამოყენება და მისი, როგორც უმთავრესი საწყისი ნაბიჯის, მნიშვნელობა პროექტზე მუშაობის დაწყებამდე.
* VS Code-ში პროექტის საქაღალდის (Folder) შექმნა და სწორი ორგანიზება.
* მონაწილეების ინდივიდუალური ტექნიკური პრობლემების მოგვარება (Troubleshooting) VS Code-ისა და Claude Code-ის ინტეგრაციისას.

**სეგმენტი B: ჩინური AI მოდელების მიმოხილვა და ლაივ-დემონსტრაციები (~დარჩენილი 30+ წუთი)**
* 7 ჩინური ხელოვნური ინტელექტის მოდელის გაცნობა: DeepSeek, Qwen, Kimi.com, Manus AI, Genspark AI, Skywork AI და Minimax.
* API კავშირის კონცეფციების ახსნა: API გასაღებები (Keys) *vs.* Computer Use (კომპიუტერის გამოყენება) *vs.* Browser Use (ბრაუზერის გამოყენება - Playwright CLI).
* პრეზენტაციებისა და ვებსაიტების გენერირების ლაივ შედარებითი დემონსტრაციები სხვადასხვა პლატფორმაზე.
* Manus AI-ის შესყიდვა Meta-ს მიერ და მისი მნიშვნელობა Meta Ads Manager-თან ინტეგრაციის კუთხით.

### 2. ძირითადი კონცეფციები და იდეები

**Claude Code-ის პროექტის გამართვის სამუშაო პროცესი (Workflow):**
* **ნაბიჯი 1:** გლობალური ინფრასტრუქტურის კონფიგურაციის ინსტალაცია (კეთდება მხოლოდ ერთხელ).
* **ნაბიჯი 2:** ახალი სესიის გახსნა და `/setup` ბრძანების გაშვება.
* **ნაბიჯი 3:** დეველოპმენტის დაწყებამდე პროექტის საქაღალდის შექმნა/მინიჭება.
* ლექტორმა შექმნა სპეციალური ფუნქცია, რომელიც `/setup` ბრძანებას საშუალებას აძლევს გააანალიზოს არსებული პროექტები, იპოვოს რა აკლია ან რა არის არასწორად კონფიგურირებული და თავიდან გამართოს იგი.
* "Ask before edits" (რედაქტირებამდე ნებართვის თხოვნა) პარამეტრის გამორთვა ("bypass") სამუშაო პროცეს
ს არსებული პროექტები, იპოვოს რა აკლია ან რა არის არასწორად კონფიგურირებული და თავიდან გამართოს იგი.
* "Ask before edits" (რედაქტირებამდე ნებართვის თხოვნა) პარამეტრის გამორთვა ("bypass") სამუშაო პროცესის დასაჩქარებლად.

**API Key vs. Computer Use vs. Browser Use:**
* **API Key (API გასაღები):** პირდაპირი პროგრამული კავშირი სერვისებს შორის (მაგ. Google AI Studio-ს დაკავშირება Claude Code-თან). დაკავშირების შემდეგ ბრაუზერი აღარ არის საჭირო — კოდი პირდაპირ ესაუბრება კოდს.
* **Computer Use:** ტექნოლოგია, სადაც AI აგენტს შეუძლია დამოუკიდებლად გახსნას ბრაუზერის ჩანართები, იაროს ვებსაიტებზე, დააჭიროს ღილაკებს და შეასრულოს მოქმედებები ავტონომიურად — API წვდომის გარეშე. ლექტორმა ეს აღწერა, როგორც "შემოვლითი გზა" API-სთან შედარებით.
* **Browser Use:** Computer Use-ის მსგავსი კონცეფცია, უბრალოდ სხვა კომპანიისგან. Claude-ს აქვს "Computer Use", ხოლო სხვა კომპანიებს აქვთ "Browser Use".
* **MCP წვდომა:** პირდაპირი წვდომის კიდევ ერთი ფორმა, რომელიც API-სთან ერთად იქნა ნახსენები.
* **Playwright CLI:** ინსტრუმენტი, რომელიც უზრუნველყოფს ბრაუზერის ავტომატიზაციას და აგენტს აძლევს საიტების გახსნის, ძიებისა და ვებ-გვერდებთან ინტერაქციის საშუალებას.

**ჩინური AI მოდელების ლანდშაფტი:**
* **DeepSeek:** უფასო, ძლიერი ტექნიკურ და კოდირების საკითხებში, ხელმისაწვდომია API-ით (ძალიან დაბალი ფასით), კარგია როგორც მეორადი ვარიანტი აგენტებისთვის. ადრე კოდირებაში საუკეთესო იყო, თუმცა ახლა ოდნავ გაუარესდა.
* **Qwen (Alibaba):** უფასო, აქვს როგორც ვებ, ისე ლოკალური ვერსიები. ითვლება მსოფლიოში ყველაზე ძლიერ ლოკალურ მოდელად. მისი უფასო ფუნქციები მოიცავს: პროექტებს, პერსონალიზაციას, სისტემურ პრომპტებს (500 სიტყვამდე), ფოტო/ვიდეო გენერირებას, ვებ-დეველოპმენტს, სლაიდებს, ღრმა კვლევას (deep research) და არტეფაქტებს.
* **Kimi.com:** ნაჩვენები იქნა რეზიუმედან ვებსაიტის შექმნის დემონსტრაციისას.
* **Manus AI:** თავდაპირველად ჩინური, რომელიც Meta-მ შეისყიდა. მთავარი შესაძლებლობა: შეუძლია დაუკავშირდეს Meta Ads Manager-ს, Instagram-ს და Instagram Creator Marketplace-ს. აქვს Computer Use ფუნქცია. აღწერილია, როგორც პრ
პირველად ჩინური, რომელიც Meta-მ შეისყიდა. მთავარი შესაძლებლობა: შეუძლია დაუკავშირდეს Meta Ads Manager-ს, Instagram-ს და Instagram Creator Marketplace-ს. აქვს Computer Use ფუნქცია. აღწერილია, როგორც პრეზენტაციების შექმნის პოტენციურად ყველაზე ძლიერი ინსტრუმენტი მსოფლიოში.
* **Genspark AI:** ასევე ნაჩვენები იქნა პრეზენტაციების შესაქმნელად, პოპულარულია მომხმარებლებში.
* **Minimax:** გთავაზობთ AI PPT (პრეზენტაციების) შექმნას, ვიდეო და აუდიო გენერირებას, მოდელის ვერსია 2.7.
* **Skywork AI:** ნახსენები იყო, თუმცა დეტალური დემონსტრაციის გარეშე.

### 3. პრაქტიკული მაგალითები და დემონსტრაციები

1. **VS Code Claude Code-ის გამართვა:** `/setup` ბრძანების ლაივ დემონსტრაცია, სადაც ნაჩვენები იყო თუ სად უნდა მოიძებნოს ის VS Code-ის ქვედა პანელში.
2. **შორენას პრობლემის მოგვარება:** ლექტორი დაეხმარა მას Claude Code-ის სწორი პანელიდან გახსნაში, ინტეგრაციის სტატუსის შემოწმებასა და არსებული პროექტების ხელახალ კონფიგურაციაში.
3. **ეკოს დახმარება VS Code-ის ინსტალაციისას:** ეკოს ჰქონდა საწყისი ეკრანის (startup screen) პრობლემა, რაც მოგვარდა კომპიუტერის გადატვირთვით (იგივე პრობლემა დაადასტურეს კობამ და მაკამაც).
4. **მაკას დახმარება ინსტალაციისას:** ლექტორი დაეხმარა მას კომპონენტის დაინსტალირებაში CLI-ში ტექსტის კოპირება-ჩასმის (copy-paste) მეთოდით, რადგან UI ღილაკი არ რეაგირებდა.
5. **Google AI Studio API Keys გვერდი:** ნაჩვენები იყო გადახდის სხვადასხვა დონე (Free tier, Tier 2 Postpay) API გასაღებებისთვის.
6. **DeepSeek-ის დემონსტრაცია:** ნაჩვენები იყო DeepThink ფუნქცია, ძიების ფუნქცია, ფაილის ატვირთვის შესაძლებლობა და კონფიდენციალურობის პარამეტრები (მონაცემების გაზიარების გათიშვა - "Improve the model for everyone").
7. **Qwen-ის დემონსტრაცია:** ნაჩვენები იყო პარამეტრები, პერსონალიზაცია, პროექტის შექმნა, ფოტო/ვიდეო გენერირება, ვებ-დეველოპმენტი და ღრმა კვლევის შესაძლებლობები.
8. **ვებსაიტების გენერირების შედარება:** ლექტორმა ატვირთა თავისი რეზიუმე Qwen-სა და Kimi.com-ზე და სთხოვა "cinematic trending dark mode" სტილის რეზიუმე-ვებსაიტის შექმნა — ნაჩვენები იყო მათი პარალელურ რეჟიმში მუშ
ტების გენერირების შედარება:** ლექტორმა ატვირთა თავისი რეზიუმე Qwen-სა და Kimi.com-ზე და სთხოვა "cinematic trending dark mode" სტილის რეზიუმე-ვებსაიტის შექმნა — ნაჩვენები იყო მათი პარალელურ რეჟიმში მუშაობის პროცესი.
9. **პრეზენტაციების გენერირების შედარება:** Manus AI-ს, Genspark AI-ს და Minimax-ს დაევალათ ქართული კომპანია "მაგთის" (MagtiCom) შესახებ დეტალური პრეზენტაციის შექმნა 2025-2026 წლების მონაცემებით.
10. **Manus AI Computer Use დემო:** ნაჩვენები იყო, როგორ მოითხოვა Manus-მა ბრაუზერის ავტორიზაცია კვლევისთვის, რის შემდეგაც ავტონომიურად მოიძია ინტერნეტში MagtiCom-ის მონაცემები და რეალური ფოტოები.
11. **SOCAR-ის პრეზენტაცია Manus AI-ით:** ნაჩვენები იყო SOCAR Georgia-ს შესახებ უკვე დასრულებული პრეზენტაცია, რომელიც შეიცავდა ინტერნეტიდან მოძიებულ რეალურ ფოტოებს, დიაგრამებს, ქართულ ტექსტს და პროფესიონალურ დიზაინს.
12. **ექსპორტის ვარიანტები:** ნაჩვენები იყო პრეზენტაციების PPTX, PDF, Google Slides, Google Drive და OneDrive ფორმატებში ექსპორტის შესაძლებლობები.

### 4. საკვანძო დასკვნები (Takeaways)

ლექტორმა მკაფიოდ გამოყო ოთხი მთავარი დასკვნა:
1. **ჩინური AI მოდელები მნიშვნელოვანია:** მონაწილეებმა დამოუკიდებლად უნდა შეისწავლონ ისინი, რადგან ისინი გვთავაზობენ ღირებულ უფასო შესაძლებლობებს, რომლებსაც შეუძლიათ შეავსონ ფასიანი ამერიკული მოდელები.
2. **წვდომის იერარქიის გაგება:** API წვდომა და MCP არის პირდაპირი კავშირები; Computer Use არის არაპირდაპირი/შემოვლითი მეთოდი. ჩინური მოდელები ვებ-ინტერფეისებზე შეიძლება ჩამოუვარდებოდნენ ამერიკულებს, მაგრამ API-ის მეშვეობით აგენტების ინტეგრაციისთვის ისინი შეიძლება ბევრად უფრო ეკონომიური და მომგებიანი იყოს.
3. **ვებ-გვერდების გენერირება:** რამდენიმე პლატფორმა გვთავაზობს ვებსაიტების გენერირებას "Playground" სტილში — მონაწილეებმა აუცილებლად უნდა იექსპერიმენტონ ამ მიმართულებით.
4. **Manus AI პრეზენტაციებისთვის:** ხაზგასმული იყო, როგორც მსოფლიოში ერთ-ერთი (თუ არა ყველაზე) ძლიერი ინსტრუმენტი პრეზენტაციების შესაქმნელად, რომელსაც აქვს ავტონომიური ვებ-კვლევისა და რეალური სურათების მოძიების უნარი.
