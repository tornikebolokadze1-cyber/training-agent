## ლექციის შეჯამება: AI ინსტრუმენტების პრაქტიკული ტრენინგი (ქართული ბიზნეს ჯგუფი)

### 1. მთავარი თემები
ეს ლექცია წარმოადგენდა პრაქტიკული AI ინსტრუმენტების კურსის მე-3 სესიას მცირე ქართული ბიზნეს ჯგუფისთვის (8-10 მონაწილე). ლექტორმა (თორნიკემ) განიხილა ექვსი ძირითადი თემატიკა: Claude Desktop App-ის ახალი ფუნქციები, პროექტის სწორი ინფრასტრუქტურა, ბრაუზერის ავტომატიზაციის ინსტრუმენტები, Claude Cowork-ის პროექტების მართვა, Google Notebook LM და Google AI Studio.

### 2. ძირითადი კონცეფციები და ახალი იდეები

**A. Claude Computer Use (ახალი ფუნქცია)**
ეს არის უმნიშვნელოვანესი ახალი შესაძლებლობა, რომელიც ლექციამდე წინა ღამით დაემატა. ხელმისაწვდომია Claude Pro და Max გეგმებზე (ლექციის მომენტში მხოლოდ Mac-ზე; Windows-ზე "რამდენიმე დღეში" გამოვა). ირთვება შემდეგნაირად: Claude Desktop App → Settings → General → Computer Use. ის საშუალებას აძლევს Claude-ს, რობოტულად მართოს თქვენი კომპიუტერი — გახსნას ფაილები, იბრაუზეროს ინტერნეტში, შეავსოს ფორმები, გადაიღოს ეკრანის ანაბეჭდები (სქრინშოტები), გაუშვას კოდი და ჩართოს აპლიკაციების ინტეგრაციები. მუშაობს როგორც ლოკალურად (კომპიუტერთან ჯდომისას), ისე დისტანციურად Dispatch ფუნქციის მეშვეობით.

**B. Dispatch ფუნქცია**
Claude Cowork-ის ფუნქცია, რომელიც სმარტფონიდან კომპიუტერის დისტანციურად მართვის საშუალებას იძლევა. პროცესი: შედიხართ Dispatch-ში Claude Cowork-დან → ასკანერებთ QR კოდს → უკავშირდებით Claude-ის მობილურ აპლიკაციას. მოითხოვს "Keep Computer Awake" (კომპიუტერის ჩართულ რეჟიმში დატოვება) და "Allow All Browser Actions" ფუნქციების გააქტიურებას. კომპიუტერი ფიზიკურად ჩართული უნდა იყოს. გამოყენების მაგალითი: შეგიძლიათ Claude-ს ნებისმიერი ადგილიდან მისცეთ დავალება, მაგალითად: "გახსენი Zoom და ავტომატურად დაგეგმე ყოველკვირეული შეხვედრები ამ 10 ადამიანთან ყოველ პარასკევს საღამოს 9 საათზე."

**C. პროექტის ინფრასტრუქტურა / ფუნდამენტი (კრიტიკული თემა)**
ლექტორმა გამოავლინა კრიტიკული პედაგოგიური პრობლემა — მონაწილეები (განსაკუთრებით შ
რები ამ 10 ადამიანთან ყოველ პარასკევს საღამოს 9 საათზე."

**C. პროექტის ინფრასტრუქტურა / ფუნდამენტი (კრიტიკული თემა)**
ლექტორმა გამოავლინა კრიტიკული პედაგოგიური პრობლემა — მონაწილეები (განსაკუთრებით შორენა) AI აგენტების პროექტებს აშენებდნენ სათანადო ფუნდამენტური სტრუქტურის გარეშე. გამოყენებული ანალოგია: "ეს ჰგავს წყალზე სახლის აშენებას — ის აუცილებლად დაინგრევა". გამოსავალი: ლექტორმა შექმნა წინასწარ გამზადებული ინფრასტრუქტურის რეპოზიტორია GitHub-ზე. მონაწილეებმა უნდა გახსნან Claude Code VS Code-ში, ჩასვან GitHub-ის ბმული და დაავალონ Claude-ს: "დააინსტალირე ეს გლობალურად ყველა პროექტისთვის". ეს ავტომატურად აგვარებს: ფოლდერების სწორ სტრუქტურას, უსაფრთხოების პარამეტრებს, .env ფაილების მართვას, Git-ის ინიციალიზაციას და GitHub-ზე ატვირთვის (push) ავტომატიზაციას.

**D. Git და GitHub-ის მარტივი ახსნა**
Git = ლოკალური ვერსიების კონტროლი (ცვლილებების ლოკალურად შენახვა/commit). GitHub = ქლაუდ (ღრუბლოვანი) სარეზერვო ასლი და საცავი. უპირატესობები: თუ კომპიუტერი გაფუჭდა, მთელი კოდი დაცულია; მუშაობის გაგრძელება შესაძლებელია ნებისმიერი კომპიუტერიდან; პროექტების გაზიარება შესაძლებელია საჯაროდ (open source) ან პირადად. ლექტორის ინფრასტრუქტურის რეპოზიტორია საჯაროა — მონაწილეებს შეუძლიათ მისი კლონირება და მასზე მუშაობა.

**E. ბრაუზერის ავტომატიზაციის ინსტრუმენტები**
ორი უფასო ინსტრუმენტი VS Code/Claude Code-ში ბრაუზერის მართვისთვის:
*   **Playwright CLI** — დამკვიდრებული ინსტრუმენტი, უფრო საიმედო ბრაუზერის ავტომატიზაციისთვის.
*   **Browser Use CLI** — ალტერნატივა ოდნავ განსხვავებული ძლიერი მხარეებით.
ორივე უფასოა და ყენდება VS Code/Claude Code-ის მეშვეობით. გამოიყენება Claude-ისთვის დავალების მიცემისას: "გამოიყენე Playwright CLI და შეასრულე ეს მოქმედებები Vercel-ზე". Computer Use (ახალი ფუნქცია) ჯერ კიდევ სუსტია ბრაუზერის ღრმა ავტომატიზაციისთვის, ამიტომ Playwright და Browser Use CLI კვლავ აუცილებელია.

**F. Claude Cowork-ის ფუნქციების ეკოსისტემა**
აიხსნა ოთხი ტიპის კასტომიზაცია:
*   **Skills (უნარები)** — სპეციალიზებული შესაძლებლობების პაკეტები, რომლებსაც ვასწავლით Claude-ს დავალებების უკეთ შესასრულებლად (მაგ. "Senior Backend დეველოპერი", "Excel-ის ანალიტიკოსი", "ბრენდის ხმის კოპირაიტერი"). იქმნება Customize → Skills → Create with Claude-ის მეშვეობით. Claude ავტომატურად იყენებს შესაბამის უნარს რელევანტური დავალებისას, პირდაპირი მითითების გარეშე.
*   **Connectors (კონექტორები)** — პირდაპირი ინტეგრაციები გარე აპლიკაციებთან (Gmail, Notion და ა.შ.).
*   **Plugins (პლაგინები)** — ნაკრები, რომელიც აერთიანებს უნარებს + კონექტორებს + ზოგჯერ ბრძანებებს.
*   **Agents (აგენტები)** — სრულად ავტონომიური, სპეციალიზებული დავალებების შემსრულებლები.
*   **Projects (პროექტები)** — გამოყოფილი სამუშაო სივრცეები Cowork-ში, საკუთარი ინსტრუქციებით, დაგეგმილი დავალებებითა და ატვირთული ფაილებით.

**G. Claude Cowork Projects — ლაივ დემონსტრაცია**
შეიქმნა პროექტი "ჩემი მოგზაურობები", რომელიც მოიცავდა: მოგზაურობის ოპტიმიზაციის კასტომიზებულ ინსტრუქციებს და დაგეგმილ დავალებას (ყოველ კვირას საღამოს 9 საათზე), რათა Claude-მა დაასკანეროს ფრენების ფასები სხვადასხვა პლატფორმაზე და გამოაგზავნოს ყოველკვირეული რეპორტი მეილზე. ნაჩვენები იყო "მოგზაურობის ფასების მონიტორის" უნარის (Skill) შექმნა AI-ის დახმარებით. ასევე სცადეს TripAdvisor-თან დაკავშირება MCP სერვერის მეშვეობით (ნაწილობრივ წარუმატებელი აღმოჩნდა სერვერის მიუწვდომლობის გამო).

**H. Skills (უნარების) დემო — ბრენდი "მთის ყავა"**
ნაჩვენები იყო წინასწარ შექმნილი ბრენდის ხმის (brand voice) უნარი "მთის ყავისთვის" — რომელიც შეიცავდა ბრენდის იდენტობას, საუბრის ტონს (თბილი, ავთენტური, მცოდნე, ამაყი) და ინსტრუქციებს კონტენტის შესაქმნელად Facebook, Instagram, LinkedIn, ელ.ფოსტის, პრესრელიზებისა და პროდუქტის აღწერებისთვის. Claude ავტომატურად იყენებს ამ უნარს, როდესაც ბრენდთან დაკავშირებული კონტენტია მოთხოვნილი.

GitHub-ზე განთავსებული ღია კოდის (open-source) ინსტრუმენტი, რომელიც აკავშირებს Claude Code-ს ლოკალურ აპლიკაციებთან: Zoom, Draw.io, LibreOffice, ინფოგრაფიკის პლატფორმები, Notebook LM და ა.შ. უფრო სტრუქტურირებულია კონკრეტული აპლიკაციების ინტეგრაციისთვის, ვიდრე Computer Use. სამომავლოდ, როცა Computer Use დაიხვეწება, ის ჩაანაცვლებს ამ ინსტრუმენტს.

**J. Google Notebook LM**
