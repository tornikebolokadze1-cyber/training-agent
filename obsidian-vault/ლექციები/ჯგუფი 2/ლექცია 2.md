## 1. მთავარი თემები

ეს იყო AI კურსის რიგით მეორე ლექცია, რომელიც ჩატარდა ქართულ ენაზე, Zoom-ის მეშვეობით. ლექტორმა (თორნიკემ) მიზანმიმართულად, კურსის საწყის ეტაპზევე (საბაზისო თემების, მაგალითად ChatGPT-ის განხილვამდე) შემოიტანა აგენტური AI-ს (Agentic AI) რთული თემები, რათა მონაწილეებს ჰქონოდათ დრო, კურსის განმავლობაში ემუშავათ საკუთარი აგენტების შექმნაზე. ლექციაზე განხილული იყო შემდეგი თემები:

*   **Claude Desktop აპლიკაცია** და მისი სამი სამუშაო რეჟიმი.
*   **Visual Studio Code (VS Code)**, როგორც პროგრამირების/დეველოპმენტის გარემო.
*   **Claude Code**, როგორც აგენტური AI კოდირების ინსტრუმენტი.
*   **MCP (Model Context Protocol) სერვერები.**
*   **ვებ-სკრეპინგის (Web scraping)** მეთოდები AI აგენტების გამოყენებით.
*   **Claude-ის Skills (უნარების)** სისტემა.
*   **API გასაღებების** მართვა და უსაფრთხოება.
*   **N8N** სამუშაო პროცესების (workflow) ავტომატიზაციისთვის.
*   **Google Antigravity (Project IDX)**, როგორც ალტერნატიული IDE.
*   **კონტექსტის ფანჯრის (Context window)** მართვა.

**გახსნითი დემონსტრაცია:** სესია დაიწყო ცოცხალი მაგალითით — Zoom-ის შეხვედრა დაგეგმა, დაიწყო და მისი ჩაწერა ავტომატურად ჩართო AI აგენტმა, ლექტორის ხელით ჩარევის გარეშე. ამან თვალსაჩინოდ აჩვენა აგენტური ავტომატიზაცია მოქმედებაში.

## 2. ძირითადი კონცეფციები და იდეები

### Claude Desktop აპლიკაცია — სამი რეჟიმი
ლექტორმა დეტალურად მიმოიხილა Claude-ის დესკტოპ აპლიკაცია (გადმოსაწერი ბმული: claude.ai/download), რომელსაც სამი განსხვავებული სამუშაო სივრცე აქვს:

2.  **Worq (CoWork):** აგენტური სამუშაო სივრცე, რომელიც დაკავშირებულია კომპიუტერის ლოკალურ ფოლდერებთან. მომხმარებელი ირჩევს ფოლდერს, აძლევს Claude-ს რთულ, მრავალეტაპიან დავალებას (მაგ: "მოი

2.  **Worq (CoWork):** აგენტური სამუშაო სივრცე, რომელიც დაკავშირებულია კომპიუტერის ლოკალურ ფოლდერებთან. მომხმარებელი ირჩევს ფოლდერს, აძლევს Claude-ს რთულ, მრავალეტაპიან დავალებას (მაგ: "მოიძიე ინფორმაცია საქართველოს ეკონომიკაზე, შექმენი Word-ის რეპორტი, ააწყვე PowerPoint-ის პრეზენტაცია, დააგენერირე სტატისტიკური ჩარტები და ყველაფერი შეინახე ამ ფოლდერში") და Claude ავტონომიურად გეგმავს და ასრულებს მას. მოიცავს **Schedule (დაგეგმვის)** ფუნქციას — დავალებების შესრულება შეიძლება დაინიშნოს ავტომატურად (ყოველდღიურად, საათობრივად, ყოველკვირეულად) მომხმარებლის ჩართულობის გარეშე (კომპიუტერი და აპლიკაცია ჩართული უნდა იყოს). ეს ფუნქცია *მხოლოდ* დესკტოპ აპლიკაციაშია და არა ვებსაიტზე.

### Visual Studio Code (VS Code)
ახსნილი იქნა, როგორც "IDE" (Integrated Development Environment - ინტეგრირებული განვითარების გარემო) — Microsoft-ის უფასო, პროფესიონალური კოდირების გარემო. გამოყენებული ანალოგია: ის არის კოდისთვის იგივე, რაც Photoshop ან Premiere Pro დიზაინისთვის/ვიდეოსთვის. მონაწილეებს დაევალათ მისი გადმოწერა. VS Code-ში მონაწილეებმა **არ უნდა გამოიყენონ** ჩაშენებული Copilot ჩატი; ამის ნაცვლად უნდა დააინსტალირონ:
*   **Claude extension** (ნარინჯისფერი აიქონი) — Claude Code-ის ინტეგრაციისთვის.
*   **Codex extension** (თეთრი ვარსკვლავის აიქონი) — ChatGPT-ის ინტეგრაციისთვის.

### ვებ-სკრეპინგის (Web Scraping) 3 მეთოდი
1.  **პირდაპირი ინტეგრაცია (საუკეთესო):** მონაცემთა წყაროების პირდაპირ დაკავშირება API-ს ან MCP-ის საშუალებით (მაგ: Gmail MCP აძლევს Claude-ს მეილებზე წვდომას). ყველაზე ორგანიზებული და ეფექტურია. *რისკი:* მონაცემები გადის Claude-ის სერვერებზე, რაც ქმნის კონფიდენციალურობის რისკებს.
2.  **Playwright MCP (ბრაუზერზე დაფუძნებული):** Claude ხსნის ბრაუზერს, შედის საიტებზე და ავტონომიურად იღებს მონაცემებს ("Computer Use"). გამოიყენება მაშინ, როცა პირდაპირი API წვდომა არ არსებობს. ნაჩვენები იყო ლაივში: Claude-მა გახსნა LinkedIn, წაიკითხა პოსტები და შეადგინა რეპორტი ადამიანის ჩარევის გარეშე. უფრო ნელი და ნაკლებად სანდოა, ვიდრე პირდაპირი ინტეგრაცია.
3.  **მესამე მხარის სკრეპინგ პლატფორმები:** სერვისები, როგორიცაა **Firecrawl AI, Epiphy, Cloudflare**, რომლებიც უზრუნველყოფენ ორგანიზებულ სკრეპინგს კონკრეტული პლატფორმებისთვის (TikTok, Google Maps, Instagram, Amazon, LinkedIn). ფასიანი სერვისებია, მაგრამ უფრო სანდო.

### Claude Skills (უნარების) სისტემა
Skills არის ინსტრუქციების ნაკრები, რომელიც ეუბნება Claude-ს *როგორ* მოიქცეს კონკრეტული დავალებების დროს (ხმის ტემბრი, სტილი, ფორმატი). არსებობს ორი ტიპი:
*   **Project Skills:** ვრცელდება მხოლოდ ერთ კონკრეტულ პროექტზე/ფოლდერზე.
*   **Global Skills:** ვრცელდება ყველა პროექტზე.
უნარების შექმნა შესაძლებელია დამოუკიდებლად (ბრენდბუქის, ძველი კონტენტის ატვირთვით) ან საზოგადოების (community) ბიბლიოთეკებიდან იმპორტირებით (მაგ: Senior Frontend Engineer უნარი, SEO ოპტიმიზატორი). ხელმისაწვდომია Claude.ai-ს skills ბიბლიო
ებლად (ბრენდბუქის, ძველი კონტენტის ატვირთვით) ან საზოგადოების (community) ბიბლიოთეკებიდან იმპორტირებით (მაგ: Senior Frontend Engineer უნარი, SEO ოპტიმიზატორი). ხელმისაწვდომია Claude.ai-ს skills ბიბლიოთეკის საიტზე.

### API გასაღებების მართვა და უსაფრთხოება
როდესაც ვქმნით აგენტებს, რომლებიც უკავშირდებიან გარე სერვისებს (Claude API, ChatGPT API, Firecrawl API), საჭიროა API გასაღებები. ლექტორმა აჩვენა გასაღებების აღება Anthropic-ისა და OpenAI-ს კონსოლებიდან. **კრიტიკული უსაფრთხოება:**
*   API გასაღებები უნდა შეინახოს **გარემოს ცვლადებში (environment variables - `.env` ფაილები)** და არასდროს არ უნდა გამოჩნდეს ღიად კოდში.
*   გაჟონილმა API გასაღებმა შეიძლება გამოიწვიოს არასანქცირებული გამოყენება და დიდი ფინანსური ზარალი.
*   API-ს გამოყენებას აქვს ცალკე ტარიფიკაცია და არ შედის სტანდარტულ ყოველთვიურ გამოწერაში.

### კონტექსტის ფანჯრის (Context Window) მართვა
Claude Code-ს აქვს კონტექსტის ფანჯარა (მისი "სამუშაო მეხსიერება" სესიისთვის). როდესაც ის ივსება (~70-80%-მდე), მუშაობის ხარისხი ეცემა. გამოსავალი: ჩაწერეთ ბრძანება `/compact`, რათა მოხდეს კონტექსტის კომპრესია და გასუფთავება. ლექტორმა აჩვენა გაფრთხილება, როდესაც კონტექსტის მხოლოდ 5% იყო დარჩენილი.

### დაგეგმვის რეჟიმები Claude Code-ში
არსებობს 4 რეჟიმი: **Plan Mode** (მხოლოდ გეგმავს), **Ask Before Edit** (ყოველი ცვლილების წინ ითხოვს ნებართვას), **Edit Automatically** (მოქმედებს გარკვეული ავტონომიით), **Bypass Permissions** (სრულიად ავტონომიური — **არ არის რეკომენდებული დამწყებთათვის**).

### N8N ინტეგრაცია
