## ლექცია #2: Claude Code, Vibe Coding და AI აგენტები — პრაქტიკული შესავალი

### 1. მთავარი თემები (Main Topics Discussed)

ეს იყო AI ტრენინგის კურსის რიგით მეორე ლექცია ქართველი ბიზნეს-პროფესიონალებისა და მენეჯერებისთვის. ლექტორმა, თორნიკე ბოლოკაძემ, მიიღო მიზანმიმართული გადაწყვეტილება, რომ პირველი თეორიული ლექციის შემდეგ პირდაპირ გადასულიყო AI აგენტების პრაქტიკულ შექმნასა და Vibe Coding-ზე (გამოტოვა რა ChatGPT-ისა და საბაზისო ხელსაწყოების შესავალი, რაც მომავალი ლექციებისთვის არის დაგეგმილი). მისი არგუმენტი: მონაწილეებს უნდა მიეცეთ კურსის სრული დრო თავიანთი აგენტების განსავითარებლად, ნაცვლად იმისა, რომ ეს თემა ბოლოსთვის შემონახულიყო.

**განხილული ძირითადი თემები:**
*   **Claude-ის ეკოსისტემა:** Chat, Cowork და Claude Code (Desktop აპლიკაცია Web ინტერფეისის წინააღმდეგ).
*   **VS Code (Visual Studio Code):** როგორც მთავარი IDE (ინტეგრირებული განვითარების გარემო) აგენტების შესაქმნელად.
*   **Google Project IDX/Antigravity:** როგორც უფასო ალტერნატიული IDE.
*   **MCP (Model Context Protocol) სერვერები:** რა არის ისინი და როგორ მუშაობენ.
*   **მონაცემთა მოპოვების (Scraping) 3 მეთოდი:** პირდაპირი API ინტეგრაცია, Playwright ბრაუზერის ავტომატიზაცია და მესამე მხარის სერვისები (როგორიცაა Apify/Firecrawl).
*   **n8n:** როგორც no-code/low-code სამუშაო პროცესების ავტომატიზაციის პლატფორმა.
*   **Claude Skills (უნარები):** Claude-ისთვის ბრენდის სპეციფიკური ქცევების სწავლება.
*   **პროექტის ინიციალიზაცია და PRD:** (Product Requirements Document - პროდუქტის მოთხოვნების დოკუმენტი) შექმნა.
*   **Claude-ის ფასები და ტოკენები:** ფასების კატეგორიები ($20, $100, $200) და მოხმარების ლიმიტები.
*   **დავალებების დაგეგმვა (Scheduling):** Claude-ის Desktop აპლიკაციაში.
*   **API გასაღებები:** და ინტეგრაციის საფუძვლები.

### 2. ძირითადი კონცეფციები (Key Concepts and Ideas Explained)

**Claude-ის სამი ინტერფეისი (Desktop აპლიკაცია):**
*   **Chat (ჩატი):** სუფთა საკომუნიკაციო არხი Claude-თან — სვამთ კითხვებს, იღებთ პასუხებს.
*   **Code (კოდი):** აგენტური კომპონენტი — შეუძლია კოდის გაშვება, აგენტების შექმნა, პროექტების აშენება, კომპიუტერის ფაილებსა და პროცესებზე წვდომა.
*   **Cowork:** შედარებით ახალი, ბუფერული ფენა, რომელიც შეიქმნა არადეველოპერებისთვის, რომლებსაც უჭირდათ Code-ის დეველოპერებზე მორგებულ ინტერფეისთან მუშაობა; ის იძლევა საქაღალდეების დონეზე მუშაობის საშუალებას ღრმა ტექნიკური ცოდნის გარეშე.
*   *მთავარი განსხვავება (რასაც ხაზი არაერთხელ გაესვა):* Desktop აპლიკაცია ბევრად უფრო ძლიერია, ვიდრე Web ინტერფეისი. Cowork და დაგეგმილი (Scheduled) დავალებები მხოლოდ Desktop-ის ფუნქციებია. Web ინტერფეისი ძირითადად მხოლოდ ჩატისთვისაა გამოსადეგი.

**IDE (ინტეგრირებული განვითარების გარემო):**
ლექტორმა ეს ახსნა ანალოგიებით: როგორც Photoshop არის ფოტოებისთვის და Premiere Pro ვიდეოსთვის — IDE არის იგივე კონცეფცია, ოღონდ კოდისთვის. VS Code (Microsoft-ისგან, უფასო) არის მთავარი რეკომენდებული ხელსაწყო; Google Antigravity/IDX არის უფასო სათადარიგო ვარიანტი.

**MCP სერვერები (Model Context Protocol):**
აიხსნა, როგორც "უნივერსალური USB კაბელი" — ნაცვლად იმისა, რომ აშენდეს ინდივიდუალური API ხიდები ყველა აპლიკაციას შორის, MCP ქმნის სტანდარტიზებულ პროტოკოლს, რომელიც Claude-ს აკავშირებს მრავალ სერვისთან ერთი თანმიმდევრული ინტერფეისით. ნაჩვენები იყო მაგალითი: Rube MCP სერვერი, რომელიც უკავშირდება Gmail-ს განსაზღვრული უფლებებითა (scopes) და ხელსაწყოებით.

**Scraping-ის (მონაცემთა მოპოვების) 3 მეთოდი:**
1.  **პირდაპირი ინტეგრაცია (საუკეთესო):** ოფიციალური API/MCP კავშირი (მაგ. Gmail MCP განსაზღვრული უფლებებით).
2.  **Playwright/ბრაუზერის ავტომატიზაცია ("ხულიგნური მეთოდი"):** AI ფიზიკურად ხსნის Chrome-ს, ნავიგაციას აკეთებს ვებსაიტებზე, იღებს სქრინშოტებს და იღებს მონაცემებს — გამოიყენება მაშინ, როცა პირდაპირი API წვდომა არ არი
ბრაუზერის ავტომატიზაცია ("ხულიგნური მეთოდი"):** AI ფიზიკურად ხსნის Chrome-ს, ნავიგაციას აკეთებს ვებსაიტებზე, იღებს სქრინშოტებს და იღებს მონაცემებს — გამოიყენება მაშინ, როცა პირდაპირი API წვდომა არ არის.
3.  **მესამე მხარის სერვისები (შუალედური):** პლატფორმები, როგორიცაა Apify, Firecrawl, Cloudflare, რომლებიც გვთავაზობენ მზა სკრეპერებს კონკრეტული პლატფორმებისთვის (TripAdvisor, Instagram, Amazon და ა.შ.).

**Vibe Coding ფილოსოფია:**
ლექტორმა ხაზგასმით აღნიშნა, რომ მონაწილეები არიან "vibe coders" — ადამიანები, რომლებმაც არ იციან კოდის წერა, მაგრამ შეუძლიათ სერიოზული პროექტების აშენება AI-ის მართვით. მაქსიმალური ტექნიკური ჩართულობა, რაც მოითხოვება, არის API გასაღებების ერთი ადგილიდან მეორეში დაკოპირება. დანარჩენი ყველაფერი სასაუბრო ფორმატში ხდება — ესაუბრებით Claude-ს ისე, როგორც მეგობარს.

**პროექტის არქიტექტურის მიდგომა:**
*   ნაბიჯი 1: პროექტის საქაღალდის შექმნა ("ყალიბი/შაბლონი").
*   ნაბიჯი 2: პროექტის სტრუქტურის ინიციალიზაცია.
*   ნაბიჯი 3: PRD-ის (პროდუქტის მოთხოვნების დოკუმენტის) შექმნა.
*   ნაბიჯი 4: დაგეგმვა ჯერ Claude-ის Web ჩატში (უფრო იაფია, ზოგავს ტოკენებს).
*   ნაბიჯი 5: გეგმის გადატანა VS Code/Claude Code-ში შესასრულებლად.
*   ნაბიჯი 6: ეტაპობრივი შესრულება კონკრეტული პრომპტებით.

**რჩევები ტოკენების ეკონომიისთვის:**
*   ყოველთვის წერეთ ინგლისურად (ქართული ენა მნიშვნელოვნად მეტ ტოკენს მოიხმარს).
*   ჯერ დაგეგმეთ Claude-ის Web ინტერფეისში, შემდეგ კი მზა გეგმები გადაიტანეთ Code-ში.
*   გამოიყენეთ სხვადასხვა მოდელები (Opus რთული დავალებებისთვის, Haiku მარტივისთვის).
*   გამოიყენეთ დაგეგმვის რეჟიმი (planning mode) შესრულების რეჟიმამდე.
*   შექმენით `memory.md` და `resume.md` ფაილები პროექტის უწყვეტობისთვის.

*   აჩვენა დაგეგმილი დავალებების (Scheduled tasks) ფუნქცია (კალენდრის ყოველდღიური შეჯამება, ევროსასამართლოს სიახლეების მონიტორინგი).

**VS Code-ის სეთაფის დემო:**
*   შექმნა ახალი საქაღალდე ("Tariel Project").
*   აჩვენა, როგორ უნდა დაყენდეს Claude და Codex ექსტენშენები (Extensions marketplace-დან).
*   აჩვენა პროექტის ინიციალიზაციის ბრძანება.
*   აჩვენა ინტერაქტიული პროცესი, სადაც Claude Code სვამს დამაზუსტებელ კითხვებს (ვებ აპლიკაცია უნდა იყოს, მობაილი თუ full-stack).

*   დააკავშირა Gmail Claude-თან Rube MCP სერვერის მეშვეობით.
*   აჩვენა ხელმისაწვდომი უფლებები (scopes) და ხელსაწყოები.
*   აჩვენა, თუ როგორ ხდება აგენტისთვის ხელმისაწვდომი ელ.ფოსტაზე წვდომა, ლეიბლები, დრაფტები და სხვა.

**Playwright/კომპიუტერის გამოყენების დემო:**
*   აჩვენა, როგორ მართავს Claude ფიზიკურად Chrome ბრაუზერს.
*   გადავიდა AI Pulse Georgia-ს Facebook გვერდზე.
*   Claude ავტონომიურად აკეთებდა ნავიგაციას, იღებდა სქრინშოტებს და აგროვებდა პოსტებს, სანამ ლექტორს ხელები აწეული ჰქონდა იმის დასამტკიცებლად, რომ ის არაფერს ეხებოდა.
