# ლექციის დეტალური შეჯამება: AI აგენტების განვითარების ინტერაქტიული სესია

## 1. მთავარი თემები

ეს სესია ჩატარდა **ინდივიდუალური კონსულტაციების ფორმატში** — ლექტორი თითოეულ მონაწილეს შეხვდა პირადად, შეაფასა პროგრესი, დაეხმარა პრობლემების მოგვარებაში და გასცა პერსონალიზებული ტექნიკური მითითებები. ტრადიციული ლექცია არ ჩატარებულა — მის ნაცვლად გამოიყენეს სოკრატული, პროექტზე ორიენტირებული მიდგომა.

**სესიის დასაწყისში შემოტანილი დამატებითი თემა:** **ChatGPT Codex**-ის დესკტოპ აპლიკაციის და მისი პლაგინების ეკოსისტემის ცოცხალი დემონსტრაცია — კერძოდ, **Hyper Frame** პლაგინი AI-ზე დაფუძნებული ვიდეო-მონტაჟისთვის და **Eleven Labs**-ის ინტეგრაცია მეტყველება-ტექსტად გადაყვანისთვის API-ს მეშვეობით.

## 2. ძირითადი კონცეფციები და განმარტებული იდეები

### A. ChatGPT Codex-ის პლაგინების ეკოსისტემა

- Codex-ს გააჩნია პლაგინების მარკეტფლეისი, რომელიც ხელმისაწვდომია ახალ ჩატში „+" ღილაკით
- დემონსტრირებული პლაგინები: **Hyper Frame** (ვიდეოს კომპოზიცია, გაყოფილი ეკრანი, სურათი-სურათში, გადასვლები, მოძრაობის ანიმაციები), **Re-motion** (გრაფიკების გენერაცია), **Hugging Face**-ის ინტეგრაცია
- Hugging Face ხელმისაწვდომია Codex-ში ჩაშენებული კონექტორის სახით — სასარგებლოა მოდელების ძიებისა და მონაცემთა ნაკრებების აღმოჩენისთვის

### B. Eleven Labs-ის API ინტეგრაცია

- Eleven Labs აღწერილია, როგორც მეტყველების AI-ის წამყვანი პლატფორმა (ტექსტი-მეტყველებად, მეტყველება-ტექსტად, ხმის იზოლაცია, დუბლირება/თარგმანი)
- API გასაღების შექმნის პროცესი ნაჩვენებია ეტაპობრივად: Developer → API Keys → Endpoints-ის შერჩევა (Speech-to-Text) → გასაღების გენერაცია
- ინტეგრაციის ძირითადი მეთოდი: Claude Code-ს ჩათში ვეკითხებით, სად ზუსტად უნდა ჩავსვათ API გასაღები (თავად ჩათში არ ჩავსვათ — უსაფრთხოების რისკია)
- გლობალური გარემოს ცვლადები პროექტის ცალკეული გასაღების განთავსებასთან შედარებით — მომავალ სესიაზე განსახილველი თემა

### C. Claude Code / Claude Desktop-ის „Skills" (უნარები)

- „Skills" — ეს არის წინასწარ შექმნილი ან საკუთრივ მორგებული, მრავალჯერ გამოსაყენებელი ინსტრუქციების ნაკრები Claude Cod
ზე განსახილველი თემა

### C. Claude Code / Claude Desktop-ის „Skills" (უნარები)

- „Skills" — ეს არის წინასწარ შექმნილი ან საკუთრივ მორგებული, მრავალჯერ გამოსაყენებელი ინსტრუქციების ნაკრები Claude Code-სა და Claude Desktop-ში
- წვდომის გზა: Customize → Skills → Plus → Create Skill → Create with Claude
- **Skills-სა და Agents-ს შორის განსხვავება:** Skills ასრულებს კონკრეტულ, განმეორებად დავალებებს; Agents კი მართავს მთლიან სამუშაო პროცესებს
- Skills-ის მარკეტფლეისი ნაჩვენებია — არსებული Skills-ები მოიცავს: Invoice Organizer-ს, კოდის ტესტირების ინსტრუმენტებს, რეფაქტორინგის ხელსაწყოებს

### D. AI პროექტების მონაცემთა ბაზის არქიტექტურა

- ლექტორმა განსაკუთრებით გაუსვა ხაზი, რომ სწორი მონაცემთა ბაზის არჩევა **ყველაზე კრიტიკული არქიტექტურული გადაწყვეტილებაა**
- განხილული სამი ტიპი:
  - **ვექტორული მონაცემთა ბაზები** — სტანდარტული, ჩაშენებაზე დაფუძნებული მოძიება
  - **გრაფ-მონაცემთა ბაზები** — კავშირებზე ცნობიერი შენახვა
  - **დროში მგრძნობიარე ცოდნის გრაფები** — მაგ., Graphiti (Zep AI-ს მიერ) — ინახავს ფაქტებს დროის კონტექსტით, „იცის", როდის შეიცვალა ინფორმაცია
- **Qdrant** წარდგენილია, როგორც ვექტორული მონაცემთა ბაზა, ხელმისაწვდომი ღრუბელშიც და ლოკალურადაც (კონფიდენციალობისთვის მგრძნობიარე გამოყენებისთვის რეკომენდებულია)

### E. მრავალი LLM-ის ორქესტრაცია

- ერთი LLM-ის გამოყენება არ არის საკმარისი რთული პროექტებისთვის
- რეკომენდებული არქიტექტურა: რამდენიმე „მუშა" LLM (ჩინური მოდელები, როგორიცაა Qwen 3.6, ინდური სამედიცინო მოდელები) → 1-2 „სინთეზატორი" LLM (Claude + ChatGPT)
- ჩინური მოდელები (Alibaba/Qwen) შექებულია ხარისხ-ფასის შეფარდებით; ზოგჯერ უფასოა
- **OpenRouter** — მოდელ-აგრეგატორი, განხილული (გაშუქდა წინა სესიებზე)

### F. RAG (მოძიებით გაძლიერებული გენერაცია)

- შემოტანილია სპეციალურად ცოდნის ინტენსიური აგენტებისთვის (აგრონომ-ბოტი, სამედიცინო კვლევის ინსტრუმენტი)
- **RAG Any** ფრეიმვორკი — მულტიმოდალური RAG, რომელიც მხარს უჭერს ტექსტს, სურათებს, ვიდეოს — არა მხოლოდ ტექსტის მოძიებას
- რეკომენდებული სამი წყაროს სინთეზი: პირადი ცოდნის ბაზა + LLM-ის ზოგადი ცოდნა +
- **RAG Any** ფრეიმვორკი — მულტიმოდალური RAG, რომელიც მხარს უჭერს ტექსტს, სურათებს, ვიდეოს — არა მხოლოდ ტექსტის მოძიებას
- რეკომენდებული სამი წყაროს სინთეზი: პირადი ცოდნის ბაზა + LLM-ის ზოგადი ცოდნა + რეალურ დროში ვებ-ძიება (Perplexity საძიებოდ)
- **Graphiti** რეკომენდებულია, როგორც RAG ძრავა დროზე დამოკიდებული ცოდნისთვის (განსაკუთრებით მაშინ, როდესაც ინფორმაცია დროთა განმავლობაში იცვლება)

### G. CLI vs. MCP (Model Context Protocol)

- CLI თანმიმდევრულად რეკომენდებულია MCP-ზე მაღლა: უფრო ღრმა წვდომა, მნიშვნელოვნად იაფი ტოკენების ხარჯვის კუთხით
- MCP „ძალიან ძვირი ღირს" (ტოკენებით), CLI კი პირდაპირი და ეფექტურია
- კონკრეტული შემთხვევები: Meta CLI — Meta MCP-ს ნაცვლად (MCP-ს ანგარიშის ხილვადობის პრობლემები აქვს), Google Workspace CLI სასურველია Analytics/Tag Manager ინტეგრაციისთვის

### H. VS Code — ძირითადი განვითარების გარემო

- კვლავ დადასტურდა, რომ ეს არის AI აგენტების შესაქმნელად საუკეთესო გარემო
- ჩაშენებული ტერმინალი გამორიცხავს ცალკე ტერმინალის ფანჯრის საჭიროებას (ხშირი საწყისი შეცდომა)
- **Cursor** და **Google Antigravity (Firebase Studio)** წარდგენილია, როგორც VS Code-ის „ფორკები" — 90%-ით იდენტური, მაგრამ VS Code რჩება მთავარ რეკომენდაციად
- Claude Code და ChatGPT Codex შეიძლება ერთდროულად გაეშვას VS Code-ში ერთ პროექტზე

### I. კონტექსტ-ინჟინერია (წინასწარ გამოცხადებული)

- გამოცხადდა, როგორც მომავალი სესიის თემა: კონტექსტის მართვა ტოკენების ზედმეტი დაწვის თავიდან ასარიდებლად

## 3. პრაქტიკული მაგალითები და დემონსტრაციები

**ცოცხალი დემო 1 — Hyper Frame ვიდეო-მონტაჟი Codex-ის მეშვეობით:**
ლექტორმა ნაჩვენა: სამი სასწავლო ვიდეოს ატვირთვა ChatGPT Codex-ში, Hyper Frame პლაგინის გააქტიურება და რთული პრომფტის შექმნა, რომელიც მოიცავდა: ქართულ ტრანსკრიფციას Eleven Labs-ის მეშვეობით, ვიდეოების გაერთიანებას, გაყოფილი ეკრანის ეფექტებს, სურათი-სურათში ჩასმას, გადასვლებს და Instagram/TikTok/Facebook-ისთვის ვირუსული სოციალური მედიის ფორმატირებას. ნაჩვენები იქნა motion design-ის ელემენტებით გამდიდრებული ვიდეო-გამოსავალი.

**ცოცხალი დემო 2 — Eleven Labs-ის API გასაღების შექმნა:**
ეტაპობრივად გაიარეს: developer-ის პორტალი, API გასაღების გენერაცია შეზღუდული წვდომებით და Codex-ში ინტეგრაცია ისე, რომ გასაღები ჩათში არ ჩაიწეროს (უსაფრთხოების პრაქტიკა).

**ცოცხალი დემო 3 — Claude Desktop Skills:**
ნავიგაცია: Customize → Skills → Browse Skills — ნაჩვენები იქნა Invoice Organizer skill და სხვა წინასწარ შექმნილი ხელსაწყოები. „Create Skill with Claude" სამუშაო პროცესი წარდგენილია, როგორც საუბრითი skill-შექმნის ინტერფეისი.

**ცოცხალი დემო 4 — Co-work / დაგეგმილი დავალებები:**

## ძირითადი თემები

- სწორი მონაცემთა ბაზის არჩევა (ვექტორული, გრაფ, ან დროში მგრძნობიარე) ყველაზე კრიტიკული არქიტექტურული გადაწყვეტილებაა AI პროექტებში.
- CLI MCP-ზე სასურველია — CLI უფრო ღრმა წვდომას იძლევა და მნიშვნელოვნად ნაკლებ ტოკენებს ხარჯავს.
- რთული AI პროექტებისთვის ერთი LLM საკმარისი არ არის — 4-10 LLM-ის კომბინაცია 'მუშა' და 'სინთეზატორის' როლებით რეკომენდებულია.
- API გასაღებები არასოდეს უნდა ჩაიწეროს პირდაპირ ჩათში — გლობალური გარემოს ცვლადები ან .env ფაილები გამოიყენეთ.
- კონფიდენციალობისთვის მგრძნობიარე მონაცემებისთვის ლოკალური განლაგება (Qdrant) ერთადერთი ნამდვილი ალტერნატივაა ღრუბელთან შედარებით.
- RAG სისტემა სამი წყაროს სინთეზით მუშაობს: პირადი ცოდნის ბაზა + LLM-ის ზოგადი ცოდნა + Perplexity-ს რეალურ დროში ძიება.
- Claude Code-ში 'Compact' ბრძანება კონტექსტი 80%-ზე ამცირებს — არ დაელოდოთ 100%-ს სანამ გამოიყენებთ.
- ვიდეო AI-ისთვის Hyper Frame + Eleven Labs + Codex კომბინაცია სოციალური მედიის კონტენტის ავტომატიზაციის მძლავრ სამუშაო ნაკადს ქმნის.
- Claude-ს ნუ მისცემთ საშუალებას, თავისი თავი ერთადერთ ვარიანტად გამოაცხადოს — ყოველთვის შეადარეთ სხვა მოდელებთან.
- Skills-სა და Agents-ს შორის განსხვავება: Skills ასრულებს კონკრეტულ, განმეორებად დავალებებს; Agents კი მართავს მთლიან სამუშაო პროცესებს.
- განვითარების პერიოდში localhost-ზე დარჩება ეფექტური — სერვერზე განლაგება მხოლოდ 15 წუთი სჭირდება მზად რომ იყოს.
- ქვე-აგენტი ხარისხის კონტროლიორის სახით ორ-აგენტის სისტემაში — ერთი ქმნის, მეორე ამოწმებს — მკვეთრად აუმჯობესებს გამოსავლის ხარისხს.
